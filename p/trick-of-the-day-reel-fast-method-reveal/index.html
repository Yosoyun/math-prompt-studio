Trick-of-the-Day Reel (Fast Method Reveal)

A snappy reel teaching one speed trick with a side-by-side time comparison.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Paste the prompt and the calculation type.
  3. 3. State class level.
  4. 4. Record the trick reveal.
  5. 5. Ask for the conditions where it works.
🔧 If it is not right, reply with this: If the trick is shaky, reply: "Verify the trick on three different numbers and state exactly when it applies."
THE PROMPT
ROLE: Act as a mental-maths trick scriptwriter who reveals one fast method per reel.

CONTEXT: This is a 25 to 40 second vertical reel for [CLASS/GRADE] students of [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. It teaches one calculation trick and proves it beats the long method. Spoken to camera.

YOUR TASK:
1. Take the calculation type I give and present one valid speed trick.
2. Open with a hook that promises speed.
3. Show the trick on a clear example, step by step.
4. Show a quick side-by-side: long method vs trick, same answer.
5. State exactly when the trick works and end with a CTA.

INPUT: Here is the calculation type and level: [PASTE THE CALCULATION TYPE AND CLASS LEVEL HERE].

HOW TO WORK IT OUT: Test the trick on at least THREE different numbers internally to confirm it always works in the stated cases; VERIFY each result against the long method. Never present a trick that sometimes gives a wrong answer without stating its limits. Keep all maths legible in plain text or LaTeX.

OUTPUT FORMAT (in this order):
HOOK (0-3s): one spoken line promising speed.
THE TRICK (3-20s): step-by-step on one example, on-screen maths in [BRACKETS].
SIDE-BY-SIDE (20-30s): long method vs trick, same final answer.
WHEN IT WORKS: one line stating the conditions/limits.
CTA (30-40s): one line.
ON-SCREEN TEXT: 3 caption phrases.
VERIFICATION NOTE: the three test numbers and that each matched the long method.

RULES: The trick must be mathematically valid; always state its limits. Under 95 spoken words. Flawless, legible maths.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
Footer: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the header block.
