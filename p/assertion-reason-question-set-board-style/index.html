Assertion-Reason Question Set (Board Style)

Board-pattern Assertion-Reason questions with the standard four-choice format and a verified explanation key.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Enter class, board and chapter.
  2. 2. Say how many A-R questions you want.
  3. 3. Send the prompt.
  4. 4. Check the four-option labels match your board.
  5. 5. Read the reason-explains-assertion column.
🔧 If it is not right, reply with this: If a Reason is not really a correct statement, reply: 'In Q3 the Reason is false - make the Reason a true statement and re-decide which of the four options applies.'
THE PROMPT
ROLE: Act as a senior CBSE-style mathematics examiner who specialises in Assertion-Reason (A-R) questions.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board]. The chapter is [CHAPTER NAME]. I want [NUMBER, e.g. 8] Assertion-Reason questions for a test.

YOUR TASK:
1. Write [NUMBER] Assertion-Reason questions from the named chapter only.
2. Each item has an Assertion (A) statement and a Reason (R) statement, both about mathematics in this chapter.
3. Use this exact four-option scheme for every question: (a) Both A and R are true and R is the correct explanation of A; (b) Both A and R are true but R is NOT the correct explanation of A; (c) A is true but R is false; (d) A is false but R is true.
4. Spread the answers across all four option types so the set is not predictable (do not make them all (a)).

INPUT: Here is the sub-topic and any specific idea I want tested: [PASTE TOPIC OR LEAVE BLANK FOR FULL CHAPTER].

HOW TO WORK IT OUT: Decide the truth of A and of R separately, with clear reasoning, before choosing the option. VERIFY each by checking the maths twice and by re-reading whether R truly explains A. Never invent a rule to make a statement true. Keep all symbols and formulas clearly written.

OUTPUT FORMAT:
First print the four-option scheme once at the top as a reference box.
SECTION 1 - QUESTIONS: For each, show 'Assertion (A): ...' then 'Reason (R): ...' then the four options (a) to (d).
SECTION 2 - ANSWER KEY: Table with columns: Q No | Correct option | Is A true? | Is R true? | Does R explain A? | One-line justification.

RULES: Both statements must be genuinely about the chapter. No vague wording. Avoid making every Reason simply repeat the Assertion. Difficulty suited to the class above.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block at the top.
