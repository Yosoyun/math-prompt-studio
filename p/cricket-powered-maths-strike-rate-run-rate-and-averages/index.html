Cricket-Powered Maths: Strike Rate, Run Rate and Averages

Cricket-themed problems that secretly teach ratios, percentages and averages.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Type the topic you want taught.
  2. 2. Enter the class and board.
  3. 3. Send and review the cricket problems.
  4. 4. Swap players' names if you wish.
  5. 5. Use answers to mark in class.
🔧 If it is not right, reply with this: If the cricket maths looks off, reply: 'Re-check every strike rate and run rate using the standard formulas, and verify each answer a second way.'
THE PROMPT
ROLE: Act as a maths teacher who is also a cricket fan and turns cricket into a maths lesson.
CONTEXT: My students are in [CLASS/GRADE] under [BOARD or EXAM e.g. CBSE / State Board]. They love cricket (IPL, India matches, gully cricket). I want to teach [TOPIC e.g. percentages / ratios / averages / data handling] through cricket so they are fully engaged.
YOUR TASK:
1. Briefly define the cricket terms you will use (strike rate, run rate, batting average, economy rate, required run rate) with their exact formulas.
2. Create 8 problems on my topic using realistic cricket scenarios and believable numbers (runs, balls, overs, wickets, targets).
3. Mix easy to hard. Include at least one 'chase' problem (required run rate) and one 'compare two players' problem.
4. Give a full solution to each, showing the formula used.
INPUT: Here is the topic and any specific sub-skill I want: [PASTE TOPIC / SUB-SKILL HERE].
HOW TO WORK IT OUT: Compute every answer yourself first. VERIFY each one in two independent ways (for example recompute strike rate as runs divided by balls times 100, then cross-check by estimating). Use only the standard cricket formulas; never invent a formula. Keep all working clean and legible. Fix any answer that fails its check before presenting.
OUTPUT FORMAT:
- 'Cricket Maths Glossary' (terms + formulas).
- 'Match-Day Problems' numbered 1 to 8 with marks.
- 'Scorecard Solutions' (full working, each ending with 'Check:' line).
- 'Coach's Note' (2 lines on where this maths appears in real cricket commentary).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
RULES: Numbers must be realistic for the format named (T20, ODI, Test). Language simple and exciting. Nothing above the class level.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
