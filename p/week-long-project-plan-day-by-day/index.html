Week-Long Project Plan (Day-by-Day)

A 5-day project schedule with daily goals, tasks, checkpoints and a final rubric.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Type the project theme and class.
  2. 2. Paste the prompt into any AI chat.
  3. 3. Read the five daily plans.
  4. 4. Share Day 1 tasks with students.
  5. 5. Use the checkpoints to track progress.
🔧 If it is not right, reply with this: If the workload is uneven, reply: "Rebalance the five days so each day has a similar workload, and keep the final-day presentation."
THE PROMPT
ROLE: Act as a maths teacher and project planner who breaks a bigger project into a clear day-by-day schedule.

CONTEXT: I teach [CLASS/GRADE] under [BOARD e.g. CBSE / ICSE / State Board]. I want a WEEK-LONG PROJECT PLAN (5 school days) on the theme [THEME e.g. budgeting a school trip / designing a garden / a statistics study / building scale models]. Students work in groups of [GROUP SIZE e.g. 4], roughly [TIME PER DAY e.g. 40 minutes] per day.

YOUR TASK:
1. State the overall project goal and final deliverable.
2. Give a day-by-day plan (Day 1 to Day 5), each with: today's goal, student tasks, and a checkpoint to verify progress.
3. Build in the maths so each day uses real mathematics from the theme.
4. Include one worked example of the key calculation students will do.
5. Make Day 5 a short presentation or display.
6. Provide a final rubric (20 marks) and a simple daily progress tracker.

INPUT: Here is my theme and time per day: [PASTE THEME AND DAILY TIME HERE]. If blank, choose a strong week-long theme for the class and board above.

HOW TO WORK IT OUT: Plan internally and ensure each day's tasks fit the stated time and build on the previous day. VERIFY the worked example two ways and show both checks. Confirm the rubric totals exactly 20. Never invent formulas. Keep all maths legible in clean plain text or simple LaTeX.

OUTPUT FORMAT (use these exact headings in this order):
1. PROJECT TITLE AND FINAL DELIVERABLE
2. OVERALL GOAL
3. DAY-BY-DAY PLAN (Day 1...Day 5, each with Goal / Tasks / Checkpoint)
4. WORKED EXAMPLE OF THE KEY CALCULATION (with two checks)
5. DAY 5 PRESENTATION FORMAT
6. DAILY PROGRESS TRACKER (a simple table)
7. FINAL RUBRIC (20 marks)
8. TEACHER TIPS

RULES: Keep each day realistic for the time given. Be concrete; no vague tasks. Short, plain sentences.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
