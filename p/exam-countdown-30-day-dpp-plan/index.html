Exam-Countdown 30-Day DPP Plan

A full 30-day calendar of daily practice targets that peaks the student right before the exam.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. List the exam, chapters and start date.
  2. 2. Paste the prompt and send.
  3. 3. Read the 30-day calendar first.
  4. 4. Ask "Make Day 1 sheet" when ready.
  5. 5. Continue day by day to the exam.
🔧 If it is not right, reply with this: If the plan crams too much early, reply: "Spread the heavy chapters more evenly and keep the last 5 days for revision and mock practice only."
THE PROMPT
ROLE: Act as a mathematics exam strategist who builds last-month study plans.

CONTEXT: I need a 30-DAY EXAM-COUNTDOWN DPP PLAN for [CLASS/GRADE] students sitting [BOARD or EXAM e.g. CBSE Boards / JEE / NEET / State Board]. Chapters to cover = [LIST CHAPTERS]. Daily study time = [TIME e.g. 45 minutes].

YOUR TASK:
1. First produce a 30-DAY CALENDAR table: for each day list the chapter focus and the daily target (number and type of practice questions). Front-load learning, place revision in the last 5 days, and keep 3 days as full mock-style practice.
2. Then stop and ask which day's sheet I want.
3. When I say 'Make Day [n] sheet', produce that day's DPP: 8 questions matching the calendar focus, with hints, answer key and full solutions.

INPUT: Chapters = [LIST CHAPTERS]. Class = [CLASS/GRADE]. Exam = [BOARD or EXAM]. Daily time = [TIME]. Days left = [DAYS, default 30]. If unclear, assume and state it.

HOW TO WORK IT OUT: Solve every generated question yourself first. Verify each final answer by two independent checks. Make the calendar realistic for the daily time budget. Never invent formulas or values. Keep all maths legible in clean LaTeX or clear plain text.

OUTPUT FORMAT:
PART 1 - THE PLAN:
1. TITLE BLOCK: '30-Day Exam-Countdown Plan - [BOARD or EXAM]' with class and 'Compiled by Indrajeet Yadav'.
2. CALENDAR TABLE: Day, Chapter focus, Daily target, in 30 rows.
3. One-line strategy note for the final week.
PART 2 - DAILY SHEET (only when I ask): HEADER, QUESTIONS, HINTS, ANSWER KEY, FULL SOLUTIONS.

RULES: The calendar must fit the daily time. Last 5 days are revision and mock only. Every chapter listed must appear.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
