Chapter Daily-Practice (DPP) Booklet over N Days -> 2 PDFs

A multi-day DPP booklet PDF plus a separate answer-key + full-solutions PDF, both compiled with pdflatex.

Works on any free AI  ·  Best tool:  ChatGPT (with code tools) or Claude

✅ How to use this effectively

  1. Fill every [SQUARE-BRACKET] placeholder (topic, days, problems per day, level, marks).
  2. Optionally paste your syllabus or a list of subtopics so the days are sequenced your way.
  3. Send it and tell it to run pdflatex and fix every log warning before replying.
  4. Download both PDFs: the DPP booklet and the answer-key + solutions.
  5. Skim Day 1 and the matching solutions to confirm the difficulty and style fit.

THE PROMPT
You are a senior Olympiad mathematics problem-setter and professional LaTeX typesetter. TASK: build a complete multi-day Daily Practice Problem (DPP) booklet for a single chapter and deliver it as TWO compiled PDFs - a clean problems booklet and a separate answer-key-plus-solutions booklet - produced with pdflatex.

PARAMETERS (the teacher fills these):
- Topic / chapter: [TOPIC]
- Class / exam target: [CLASS/EXAM e.g. JEE Advanced / RMO / INMO / Putnam]
- Number of days: [DAYS]
- Problems per day: [PROBLEMS PER DAY]
- Subtopics or sequence (optional): [CHAPTERS/SUBTOPICS]
- Difficulty spread per day: [e.g. 2 warm-up, 2 core, 1 challenge]
- Marks per problem / total marks per day: [TOTAL MARKS]
- Booklet title: [TITLE]
- Footer text: Indrajeet Yadav

PHASE 1 - PLAN. From the parameters, design a coherent [DAYS]-day arc for [TOPIC] at [CLASS/EXAM] level. If [CHAPTERS/SUBTOPICS] is given, distribute subtopics so each day has a clear focus and the difficulty rises across the days. If it is blank, infer the standard subtopics of [TOPIC] and sequence them from foundational techniques to multi-idea hybrids. State (to yourself only) the learning aim of each day; do not pad the booklet with commentary.

PHASE 2 - DRAFT. Write exactly [PROBLEMS PER DAY] original problems for each of the [DAYS] days following the difficulty spread. Demand insight, not mechanical substitution: every problem should hinge on a genuine idea, a case split, an extremal argument, an invariant, or a clever transformation. No two problems may be near-duplicates. Avoid tired textbook clones. Mix question types (find-all, prove-that, evaluate exactly, maximise/minimise) appropriately for [TOPIC].

PHASE 3 - SOLVE AND VERIFY. Independently solve EVERY problem yourself; never trust a value you wrote in Phase 2. Use exact values (surds, fractions, pi, logs) - never decimals as final answers. For find-all problems prove BOTH necessity and sufficiency. For inequalities give the equality condition. Handle every case and boundary; check domains and excluded values. No circular reasoning, no 'clearly', no fabricated theorems or statistics. If a problem turns out flawed or ambiguous, replace it and re-solve. Where feasible, sanity-check numerics in code before trusting them.

PHASE 4 - TYPESET AND COMPILE. Produce two LaTeX documents, each documentclass[11pt]{article}, default Computer Modern font, permitted packages only: amsmath, amssymb, enumitem, geometry (margin=1in), array, fancyhdr. Footer via:
```
\pagestyle{fancy}
\fancyhf{}
\fancyfoot[C]{Indrajeet Yadav}
\renewcommand{\headrulewidth}{0pt}
\renewcommand{\footrulewidth}{0pt}
```
Use \dfrac, \left/\right, \lfloor/\lceil, \binom, \pmod, \mathbb, display math with \[ ... \] and aligned/cases. Never use a Unicode math symbol when a LaTeX command exists. Number problems Day 1: 1.1, 1.2, ...; Day 2: 2.1, ...; show marks beside each. Solutions are labelled S1.1, S1.2, ... matching the problem numbers.

DESIGN RULES: strict black-and-white minimalist. No colours, no boxes, no shading, no borders, no icons, no logos, no watermarks, no difficulty stars, no banners, no decorative rules. Clean headings, generous spacing, nothing ornamental.

TWO PDFs:
- PDF 1 = the DPP booklet: title, then each day as a section with its problems and marks ONLY. No answers, no hints, nothing that leaks a solution.
- PDF 2 = answer key + solutions: first a compact answer-key table (problem number -> final answer) for every day, then full rigorous solutions S1.1, S1.2, ... each showing the key idea and complete reasoning to the exact answer.
Filenames: PDF 1 = DPP_[TOPIC]_Indrajeet_Yadav_Problems.pdf and PDF 2 = DPP_[TOPIC]_Indrajeet_Yadav_Solutions.pdf (replace [TOPIC] with a short slug, no spaces).

COMPILE: run pdflatex on each file, open and read the log, and fix every error, every overfull/underfull box, every broken formula and every bad page break (ideally start each day on a clean page). Recompile until both logs are clean, then visually check every page top to bottom.

HONESTY: never invent theorems, statistics, or named results. If any supplied parameter is contradictory or unclear, ask one brief clarifying question before building; reconstruct intended meaning only when it is unambiguous.

FINAL DELIVERY: reply with ONLY the two downloadable PDF links plus one short sentence confirming both compiled cleanly with pdflatex. Do NOT reproduce the problems, the answer key, the solutions, or any LaTeX source in the chat.
