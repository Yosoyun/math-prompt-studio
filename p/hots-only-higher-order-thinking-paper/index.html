HOTS-Only Higher-Order-Thinking Paper

A paper made entirely of higher-order-thinking problems that stretch your strongest students.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Paste the prompt into a chat.
  2. 2. List the chapters to draw from.
  3. 3. Set the class and marks.
  4. 4. Send and read the tougher items.
  5. 5. Study the worked key for each answer.
  6. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If a question is just routine, reply: 'Question [number] is too routine for a HOTS paper. Replace it with a multi-step or reasoning-heavy question on the same concept, then provide its full worked solution.'
THE PROMPT
ROLE: Act as a mathematics teacher who designs higher-order-thinking (HOTS) challenge papers.

CONTEXT: I need a HOTS-ONLY paper for Class [CLASS/GRADE], [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE], to stretch my strongest students. Every question must require analysis, multi-step reasoning, application or proof - no plain recall or one-step sums. Total marks [TOTAL MARKS], time [TIME].

YOUR TASK:
1. Write questions that combine concepts, demand reasoning, or have an unexpected twist.
2. Include some 'explain why' or 'prove that' items where suitable.
3. Keep them solvable within the class syllabus - hard, but fair.
4. Provide a fully worked solution key, not just final answers.

INPUT: Chapters or topics to draw from: [LIST CHAPTERS or TOPICS]. Use only these.

HOW TO WORK IT OUT: Solve every question internally with full working before printing. Verify each final answer twice by independent methods. Confirm each question genuinely needs higher-order thinking and is still within syllabus. Never invent formulas or values. Keep all mathematics legible in clean LaTeX or plain text. Confirm marks total exactly [TOTAL MARKS].

OUTPUT FORMAT, in this order:
1. HEADER: 'HOTS Challenge Paper - Mathematics', class, time, maximum marks, 'Compiled by Indrajeet Yadav'.
2. INSTRUCTIONS: 3 to 4 numbered lines, warning that all questions are challenging.
3. THE PAPER: numbered questions, marks beside each.
4. WORKED SOLUTION KEY: full step-by-step solution for every question.

RULES: No routine recall questions. Stay inside the listed syllabus. Hard but solvable. Number every question. Marks total exactly.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
