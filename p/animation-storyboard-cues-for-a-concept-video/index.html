Animation Storyboard Cues for a Concept Video

A frame-by-frame storyboard with narration, visuals, and on-screen text cues.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Paste the prompt and your concept.
  3. 3. State video length and class level.
  4. 4. Hand the scenes to your editor.
  5. 5. Ask for simpler visuals if needed.
🔧 If it is not right, reply with this: If visuals are vague, reply: "Describe each visual concretely - exact shapes, labels, colours and motion - so an animator needs no guesswork."
THE PROMPT
ROLE: Act as a storyboard director for animated mathematics explainer videos.

CONTEXT: A maths teacher will hand this storyboard to an editor or simple animation tool to build a concept video for [CLASS/GRADE] students of [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. They need scene-by-scene cards: what is narrated, what is shown, and what text appears.

YOUR TASK:
1. Take the concept I provide and split the video into 6 to 10 scenes.
2. For each scene give: narration (VOICEOVER), the VISUAL to draw or animate, and ON-SCREEN TEXT.
3. Describe each visual concretely (exact shapes, labels, arrows, motion) so an animator needs no guesswork.
4. Include at least one worked example animated step by step.
5. End with a recap scene and a CTA scene.

INPUT: Here is the concept, length and level: [PASTE CONCEPT, TARGET LENGTH AND CLASS LEVEL HERE].

HOW TO WORK IT OUT: Solve any example fully first and VERIFY the answer two independent ways. Never invent a formula or a misleading diagram; every labelled visual must be mathematically accurate. Keep all maths legible in plain text or LaTeX for the on-screen text.

OUTPUT FORMAT (numbered scene cards, in this order):
VIDEO TITLE: one line.
For each scene: Scene N - VOICEOVER: "..." | VISUAL: concrete description | ON-SCREEN TEXT: "..." | DURATION: seconds.
Scenes must include: Hook, Concept setup, Worked example (one scene per step), Common mistake, Recap, CTA.
TOTAL RUNTIME: sum of durations.
VERIFICATION NOTE: how the example was checked two ways.

RULES: Visual descriptions must be specific and accurate. Durations should sum to the target length. Maths flawless and legible. No vague directions like "show some maths".


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
Footer: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title block.
