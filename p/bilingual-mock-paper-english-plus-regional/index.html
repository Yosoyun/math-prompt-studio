Bilingual Mock Paper (English plus Regional)

Each question in English and your chosen language, side by side, with one key.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Name the second language you need.
  2. 2. List chapters and total marks.
  3. 3. Send the prompt.
  4. 4. Ask to fix any awkward translation.
  5. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If a translation reads oddly, reply: "Re-translate question 2 into natural exam-style [LANGUAGE]; keep all numbers, symbols and formulas identical to the English."
THE PROMPT
ROLE: Act as a bilingual examination paper-setter who writes mathematically precise dual-language papers.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. State Board / CBSE]. Many students read better in [SECOND LANGUAGE e.g. Hindi / Marathi / Tamil / Telugu / Bengali]. I need a mock where every question appears in BOTH English and [SECOND LANGUAGE], so no student is disadvantaged by language.

YOUR TASK:
1. Build a full mock for these chapters: [LIST CHAPTERS]. Total marks: [TOTAL MARKS]. Time: [TIME].
2. For EACH question, present the English version first, then the [SECOND LANGUAGE] version immediately below it, clearly separated. The two versions must be exactly equivalent in meaning and difficulty.
3. Keep all numbers, mathematical symbols, variables and formulas IDENTICAL in both languages - only the words change. Use natural, exam-style [SECOND LANGUAGE].
4. Provide ONE solution set and one answer key (the maths is language-independent).

INPUT: Here are the chapters, marks and the second language: [PASTE DETAILS HERE].

HOW TO WORK IT OUT: Solve everything internally first. VERIFY each answer two independent ways. Never invent formulas or values. Confirm the two language versions of each question ask exactly the same thing. Keep all maths legible in clean plain text or LaTeX, identical across both languages.

OUTPUT FORMAT (exact order):
HEADER: bilingual title block, Subject, Class, Board/Exam, Marks, Time, Instructions in both languages.
PAPER: for each numbered question -> 'EN:' line, then '[SECOND LANGUAGE]:' line, then options if MCQ (also in both).
SOLUTIONS: full steps (English).
ANSWER KEY: compact list.
VERIFICATION NOTE: confirm answers checked and both language versions match.

RULES: Identical maths in both languages; natural translation. Marks total [TOTAL MARKS]. Original questions. Difficulty true to [BOARD or EXAM].

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
