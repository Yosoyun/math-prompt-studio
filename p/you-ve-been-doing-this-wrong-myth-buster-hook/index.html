"You've Been Doing This Wrong" Myth-Buster Hook

A myth-busting short that corrects a common math mistake with proof.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Paste the prompt and the mistake.
  3. 3. State class level.
  4. 4. Record the wrong-then-right reveal.
  5. 5. Ask for a softer tone if too harsh.
🔧 If it is not right, reply with this: If the correction is unclear, reply: "Show the wrong method and the right method side by side with the same numbers and verify the right answer."
THE PROMPT
ROLE: Act as a myth-busting maths content writer who corrects common student errors with proof.

CONTEXT: This is a 30 to 50 second vertical short for [CLASS/GRADE] students of [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. The hook is the classic "you have been doing this wrong" pattern: name a common mistake, show why it is wrong, then teach the correct way. Spoken to camera.

YOUR TASK:
1. Take the common mistake I give and open with a confident "you have been doing this wrong" style hook (give 3 hook options).
2. Show the WRONG method on a sample problem and the wrong answer it produces.
3. Show the RIGHT method on the SAME problem and the correct answer.
4. Explain in one line WHY the wrong method fails.
5. End with a memorable correction the student will not forget.

INPUT: Here is the common mistake and level: [PASTE THE MISTAKE AND CLASS LEVEL HERE].

HOW TO WORK IT OUT: Work both the wrong and right methods fully on the same numbers. VERIFY the correct answer by two independent methods. Never misrepresent the wrong method as a straw man; show the real mistake students actually make. Keep all maths legible in plain text or LaTeX.

OUTPUT FORMAT (in this order):
HOOK OPTIONS: 3 spoken lines.
THE WRONG WAY (5-20s): spoken + on-screen maths in [BRACKETS], ending in the wrong answer.
THE RIGHT WAY (20-40s): spoken + on-screen maths in [BRACKETS], ending in the correct answer.
WHY IT FAILS: one line.
REMEMBER (40-50s): the memorable correction.
ON-SCREEN TEXT: 4 caption phrases.
VERIFICATION NOTE: how the correct answer was checked two ways.

RULES: Be firm but kind, never mocking. Same numbers in both methods. Under 120 spoken words. Flawless, legible maths.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
Footer: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the header block.
