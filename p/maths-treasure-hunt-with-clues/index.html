Maths Treasure Hunt With Clues

An unbroken clue trail where each answer decodes the next hiding spot, with an answer key and hint envelopes.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Type your topic, class and board.
  2. List your real school locations.
  3. Paste the prompt and send.
  4. Hide each clue at its decoded spot.
  5. Keep hint envelopes ready at each station.
🔧 If it is not right, reply with this: If an answer does not point cleanly to a place, reply: "Re-check the answer-to-location key, re-verify each clue, and trace the trail again so there are no loops or dead ends."
THE PROMPT
ROLE: Act as a mathematics teacher and treasure-hunt designer who builds a clue-to-clue maths trail around the classroom or school.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. I want a maths treasure hunt on [TOPIC] where solving each problem reveals WHERE the next clue is hidden, ending at a final prize spot.

YOUR TASK:
1. Design a trail of [NUMBER, e.g. 8] clue stations around common school locations (library, lab, notice board, water cooler, etc.).
2. At each station, a maths problem on [TOPIC] yields an answer that decodes the next location (give me a simple answer-to-location key, for example answer 12 means 'go to room 12' or maps to a letter).
3. Provide the master location map and the teacher answer key.
4. Add a tamper rule so teams cannot just follow each other, and a hint envelope per station.

INPUT: Topic, class, board and number of stations are above. List your real locations here if you like: [PASTE LOCATIONS], otherwise use generic ones.

HOW TO WORK IT OUT: Solve every clue, then trace the full trail to confirm each answer correctly points to the next station with no loops or dead ends. Verify each answer twice by independent methods. Never invent formulas or values. Keep all maths in clean LaTeX or clear plain text.

OUTPUT FORMAT (use these exact headings):
THE STORY (2-3 lines)
TEACHER SETUP (where to hide each clue, numbered)
ANSWER-TO-LOCATION KEY (how answers decode to places)
CLUE STATIONS 1 to N (the problem students see at each)
MASTER ANSWER KEY (each answer in a box [Answer: ...] and the place it reveals)
HINT ENVELOPES (one graded hint per station)
ANTI-COPYING RULE
TRAIL CHECK (confirm the path is unbroken)

RULES: The trail must be a single unbroken path. Answers must decode cleanly to locations. No two clues may repeat the same step. Keep clue text readable for a teenager.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
