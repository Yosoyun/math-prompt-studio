Solve and Flag If the Question Has an Error

An honest check of whether the printed question is flawed, plus a solution or a fix.

Attach a photo of the question first  ·  Best tool:  ChatGPT or Gemini (image)

✅ How to use this effectively

  1. 1. Photograph the question you suspect is faulty.
  2. 2. Attach the photo and paste this prompt.
  3. 3. Send and read the verdict.
  4. 4. If flawed, read the suggested correction.
  5. 5. Re-photograph if the AI misread it.
🔧 If it is not right, reply with this: If you disagree with the verdict, reply: "Re-read the photo exactly. Assume nothing is missing. Re-check whether the data is consistent, then re-decide."
THE PROMPT
ROLE: Act as a rigorous mathematics examiner who checks whether a printed question is even correct before solving it.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET], topic [TOPIC]. I suspect the question in my photo may have a misprint, missing data, or an impossible setup. Tell me the truth.

YOUR TASK:
1. Read the question in the attached photo and restate it exactly in clean text.
2. Check whether it is well-posed: are the given data consistent, is enough information provided, is the answer well-defined, are there contradictions or typos?
3. Decide: SOLVABLE AS WRITTEN, or HAS A PROBLEM.
4. If solvable, solve it fully. If flawed, describe the exact flaw and offer the most likely intended question, then solve that corrected version.

INPUT: I have attached a photo of the problem. If a symbol is unclear, tell me which one and your best reading - but do NOT label a question flawed just because of your own misreading; flag genuine inconsistencies only.

HOW TO WORK IT OUT: Try honestly to solve the question as written. If it leads to a contradiction, impossible value, or undefined result, investigate whether that is due to a flaw in the question. VERIFY any final answer with TWO independent checks. Never invent missing data to force a solution; instead state clearly what is missing. Name standard results correctly. Keep all mathematics fully legible in clean LaTeX or clear plain text, one step per line.

OUTPUT FORMAT (use these exact headings):
1. Question as I read it.
2. Well-posed check - bullet points on consistency, sufficiency, and definedness.
3. Verdict - exactly one of "SOLVABLE AS WRITTEN" or "HAS A PROBLEM", with a one-line reason.
4. If flawed - the exact flaw, and the most likely intended question.
5. Solution - of the original (if valid) or the corrected version, numbered steps.
6. Final answer - clearly stated, or "No valid answer - reason: ...".
7. Verification - Check 1 and Check 2, where applicable.

RULES: Be honest - do not pretend a broken question works. Do not invent data. Use plain ASCII only with straight quotes. Be exact unless an estimate is requested.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block at the top.
