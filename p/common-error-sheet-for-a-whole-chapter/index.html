Common-Error Sheet for a Whole Chapter

A printable sheet of the top mistakes students make in a chapter, each with the fix.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Type the chapter or topic name.
  2. Set the class and board or exam.
  3. Choose how many errors you want.
  4. Press Enter to get the sheet.
  5. Ask to add chapter-specific examples.
🔧 If it is not right, reply with this: If it is too generic, reply: "Add a worked wrong-vs-right example for each error from [CHAPTER], and verify each correct answer twice."
THE PROMPT
ROLE: Act as an experienced examiner compiling the classic mistakes for one chapter.

CONTEXT: This is for [CLASS/GRADE] students studying the chapter [CHAPTER NAME e.g. Trigonometric Identities] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE]. I want a single reference sheet of the errors that cost my students the most marks, so I can pre-empt them.

YOUR TASK:
1. List the [NUMBER e.g. 10] most common and most costly errors students make in this chapter.
2. For each error, show a short WRONG example and the RIGHT correction side by side.
3. Add the underlying misconception and a one-line memory tip.
4. End with a quick self-test the student can use.

INPUT: Here is the chapter and level: [CHAPTER NAME], [CLASS/GRADE], [BOARD or EXAM]. If you want me to focus on certain sub-topics, I will add them here: [OPTIONAL SUB-TOPICS].

HOW TO WORK IT OUT: For every worked example, solve it internally and VERIFY the correct version with two independent checks before printing it. Choose errors that genuinely recur at this level, not rare edge cases. Never invent formulas or values; name standard results. Keep all maths legible in clean LaTeX or plain text.

OUTPUT FORMAT:
Title block: 'Common Errors - [CHAPTER NAME] ([CLASS/GRADE], [BOARD or EXAM])'.
Then a numbered list; each item uses these labels:
- Error: short name.
- Wrong: the tempting incorrect line.
- Right: the corrected line.
- Why it happens: the misconception in one sentence.
- Tip: one memory hook.
Finish with 'Quick Self-Test' - five short spot-the-error questions and an answer key (with verification noted).

RULES: Plain ASCII only, straight quotes. Keep each item tight enough to fit a printed page; aim for clarity a busy student can scan.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
