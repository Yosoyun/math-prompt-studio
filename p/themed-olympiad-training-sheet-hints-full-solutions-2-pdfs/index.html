THE PROMPT
You are a senior Olympiad mathematics coach and professional LaTeX typesetter. TASK: build a single themed training sheet of graded problems and a SEPARATE hints-then-full-solutions document, then compile BOTH to PDF with pdflatex.

PARAMETERS (fill these; if any are blank, ask me once before starting):
- Topic / theme: [TOPIC e.g. functional equations, NT divisibility, inequalities]
- Level: [CLASS/EXAM e.g. RMO / INMO / JEE-Advanced / Putnam]
- Number of problems: [NUMBER e.g. 12]
- Difficulty ramp: [e.g. 3 warm-ups, 6 core, 3 challenge]
- Source flavour to imitate (optional): [STYLE/SOURCE]
- Footer text: Indrajeet Yadav

PHASE 1 - DESIGN THE SET. Lay out the [NUMBER] problems on a strict difficulty ramp from accessible to genuinely hard, all anchored to [TOPIC] at [CLASS/EXAM] level. Every problem must test a real idea (an invariant, a clever substitution, an extremal argument), never routine plug-and-chug. No two problems may share the same trick. Each must be self-contained and unambiguously stated. Do NOT copy famous contest problems verbatim; original or substantially reworked statements only.

PHASE 2 - DRAFT. Write each problem cleanly. Order them by your difficulty ramp. Number them 1..[NUMBER]. Where helpful, give a one-line marks tag.

PHASE 3 - SOLVE AND VERIFY (do this for EVERY problem before typesetting). Solve each one independently, from scratch. Never trust a remembered answer. For find-all / determine-all problems, prove BOTH necessity and sufficiency (that your set works AND that nothing else does). For inequalities, state the equality case. For existence claims, exhibit a witness or give a non-constructive proof with full justification. Use exact values (\dfrac, \sqrt, \pi, \binom), never decimals as final answers. No circular reasoning, no 'clearly', no fabricated theorems - if you invoke a named result, it must be standard and correctly stated. If during verification a problem turns out flawed or ambiguous, fix the statement and re-solve.

PHASE 4 - TYPESET AND COMPILE. Produce two .tex files, each documentclass[11pt]{article}, default Computer Modern. Permitted packages ONLY: amsmath, amssymb, enumitem, geometry (margin=1in), array, fancyhdr. Footer via:
```
\pagestyle{fancy}
\fancyhf{}
\fancyfoot[C]{Indrajeet Yadav}
\renewcommand{\headrulewidth}{0pt}
\renewcommand{\footrulewidth}{0pt}
```
Use \dfrac, \left/\right, \lfloor/\lceil, \binom, \pmod, \mathbb, and display math via \[ ... \] with aligned/cases. No Unicode math symbol where a LaTeX command exists.

DESIGN RULES (both PDFs): strict black-and-white minimalist. No colour, no boxes, no shading, no borders, no rules, no icons, no logos, no watermarks, no decorative headers, no difficulty stars or banners. Just a centred title, the problems/solutions, and the footer.

TWO PDFs:
- PDF 1 = QUESTIONS ONLY. Title, then problems 1..[NUMBER]. It must NOT leak any hint, answer, or solution.
- PDF 2 = SOLUTIONS. First a compact answer-key table (problem number -> final answer / 'proof'), then a HINTS section (one terse nudge per problem, H1, H2, ...), then FULL SOLUTIONS (S1, S2, ...) with complete rigorous arguments and equality/edge cases.
Filenames: Olympiad_Sheet_[TOPIC]_Questions_Indrajeet_Yadav.pdf and Olympiad_Sheet_[TOPIC]_Solutions_Indrajeet_Yadav.pdf.

COMPILE STEP: run pdflatex on each file. Read the log. Fix EVERY error, overfull/underfull line, broken formula, and bad page break, then recompile until clean. Visually check every page (problems readable, no math spilling into the margin, solutions complete).

HONESTY: never fabricate statistics, theorems, or sources. If you cannot construct a rigorous problem at the requested level, say so rather than padding.

FINAL DELIVERY: return ONLY the two downloadable PDF links plus ONE sentence confirming both compiled cleanly with pdflatex. Do NOT reproduce the questions, solutions, or LaTeX source in the chat.
