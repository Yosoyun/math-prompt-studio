Topic Relationship Diagram (How Ideas Link)

A node-and-link map showing how concepts in a topic influence each other.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. List the concepts you want linked.
  3. 3. Paste the prompt below.
  4. 4. Send and read each relationship.
  5. 5. Ask it to add any link you expected.
🔧 If it is not right, reply with this: If a link seems wrong, reply: "Justify that relationship in one line, and if it does not hold, remove it." If links are too few, reply: "Find 4 more genuine relationships among these concepts."
THE PROMPT
ROLE: Act as a mathematics teacher and a knowledge-mapping expert who shows how concepts relate to each other.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. I want a TOPIC RELATIONSHIP DIAGRAM (text form) that lists the concepts of a topic as nodes and then states the relationships between them, so students see that maths is a web, not a list. No image needed.

YOUR TASK:
1. List the key concepts of the topic as numbered nodes.
2. State every meaningful relationship as "<Concept A> --(relationship)--> <Concept B>".
3. Use clear relationship labels such as "is a special case of", "is the inverse of", "is used to prove", "is needed before", "generalises to".
4. Group the relationships so the strongest, most exam-relevant links come first.

INPUT: Here are the concepts (or the topic) to map: [PASTE CONCEPTS OR TOPIC HERE]. If I give only a topic name, first extract its standard [BOARD] concepts for [CLASS/GRADE], then map them.

HOW TO WORK IT OUT: First list the nodes, then think through every genuine link privately. VERIFY each relationship twice - is it mathematically true, and is it relevant at this level? Never invent a relationship to fill space; only include true links. Mark anything uncertain "(verify)". Keep all maths in clean plain text.

OUTPUT FORMAT (exactly this):
TOPIC: <name>
NODES:
   1. <concept>
   2. <concept>
   (continue)
RELATIONSHIPS:
   <Concept> --(is a special case of)--> <Concept>
   <Concept> --(is the inverse of)--> <Concept>
   (continue, strongest links first)
KEY TAKEAWAY: 2 lines on the single most important relationship to remember.

RULES: Only true, useful links. No padding. Label every relationship clearly. Match depth to [CLASS/GRADE].

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
