Blueprint-Driven Paper from My Own Weightage Table

A paper built to hit your exact chapter weightage and question-type counts, verified cell by cell.

Works on any free AI  ·  Best tool:  ChatGPT (free) or Claude (free)

✅ How to use this effectively

  1. 1. Write your blueprint as a simple table.
  2. 2. Paste it under INPUT exactly.
  3. 3. Read the coverage check it returns.
  4. 4. Compare its table to yours.
  5. 5. Copy into Word and export PDF.
  6. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If a chapter's marks are off, type: 'Chapter [X] should carry [Y] marks but currently carries [Z]. Adjust questions so each chapter matches my blueprint exactly, then reprint the coverage table.'
THE PROMPT
ROLE: Act as an examination blueprint specialist who constructs papers that match a given weightage matrix to the mark.

CONTEXT: I teach [CLASS/GRADE] [SUBJECT] for [BOARD/EXAM]. Total marks: [TOTAL MARKS]. Duration: [DURATION]. I will give you a blueprint table telling you how many marks each chapter must carry and how those marks split across question types (MCQ / short / long / case-study). The paper MUST obey this blueprint exactly.

YOUR TASK:
1. Read my blueprint and design a single complete paper whose chapter-wise marks and question-type counts match every cell.
2. Add a header block, general instructions, sections, and marks in brackets.
3. Tag each question discreetly with its chapter in square brackets at the end of the question line, e.g. [Ch: Trigonometry], so coverage is auditable.
4. Produce a final 'Blueprint Verification' table: chapter, target marks (from me), actual marks (in the paper), and a Match/Mismatch flag per row.

INPUT: Here is my blueprint table:
[PASTE TABLE - e.g. Chapter | Marks | MCQ | Short | Long | Case]
Use ONLY the listed chapters and respect every number.

HOW TO WORK IT OUT: Solve each question silently to ensure solvability. VERIFY by two independent checks: (a) sum each chapter's actual marks and compare to my target - every row must read Match; (b) sum all chapters and confirm it equals [TOTAL MARKS], and confirm each question-type count matches. If any cell cannot be met exactly, stop and tell me the conflict instead of fudging. Never invent out-of-syllabus content or formulas. Keep mathematics legible: fractions a/b, powers x^2, roots sqrt(x).

OUTPUT FORMAT (exact order):
1. Header block.
2. General Instructions.
3. Full paper, each question tagged with its chapter.
4. Blueprint Verification table (target vs actual vs Match/Mismatch).
5. Note: 'All rows Match; grand total verified = [TOTAL MARKS]'.

RULES: The blueprint is non-negotiable; report conflicts rather than guess. No answer key here. Plain formatting for clean Word/PDF export.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
