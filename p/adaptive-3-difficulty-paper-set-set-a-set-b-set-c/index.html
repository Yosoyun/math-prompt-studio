Adaptive 3-Difficulty Paper Set (Set A / Set B / Set C)

Three parallel papers on the same syllabus at easy, standard and challenge levels for differentiated classes.

Works on any free AI  ·  Best tool:  Claude (free) or ChatGPT (free)

✅ How to use this effectively

  1. 1. State chapters, marks and duration once.
  2. 2. Ask for Set A/B/C, same blueprint.
  3. 3. Confirm all three cover identical topics.
  4. 4. Copy each set into its own Word file.
  5. 5. Export each as a separate PDF.
  6. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If the levels feel too similar, type: 'Increase the gap: make Set A purely single-step, Set B two-step, and Set C multi-step or proof-based, keeping the same topics and marks.'
THE PROMPT
ROLE: Act as an assessment designer who builds parallel differentiated exam papers for mixed-ability classrooms.

CONTEXT: I teach [CLASS/GRADE] [SUBJECT] for [BOARD/EXAM]. I need THREE papers on the same syllabus and same blueprint but three difficulty levels: Set A (foundation/easy), Set B (standard/board level), Set C (challenge/advanced). Each paper: total marks [TOTAL MARKS], duration [DURATION].

YOUR TASK:
1. Fix one common blueprint (same number of questions, same sections, same marks per question across all three sets) so the sets are truly parallel.
2. Generate Set A, Set B, Set C, each covering the SAME topics in the SAME order, differing ONLY in cognitive demand: Set A single-step recall/application, Set B two-step standard exam, Set C multi-step / reasoning / proof-style.
3. Give each set its own header block and marks in brackets.
4. After all three, add a short comparison table: question number, topic, and the level-A/B/C task for that slot, so I can see they are matched.

INPUT: Topics/chapters to use: [PASTE CHAPTERS]. Use ONLY these across all three sets.

HOW TO WORK IT OUT: Solve every question in all three sets silently to confirm solvability and that difficulty truly rises A < B < C. VERIFY by two checks: (a) the three sets share identical blueprint and total = [TOTAL MARKS] each; (b) for each matched slot, the topic is the same across sets but the number of solution steps increases from A to C. Never go out of syllabus or invent formulas. Keep mathematics legible: fractions a/b, powers x^2, roots sqrt(x).

OUTPUT FORMAT (exact order):
1. Common Blueprint summary (one short block).
2. SET A - foundation: header + full paper.
3. SET B - standard: header + full paper.
4. SET C - challenge: header + full paper.
5. Matched-slot comparison table.
6. Note: 'Each set verified total = [TOTAL MARKS]'.

RULES: Sets must be genuinely parallel (same coverage), not random variants. No answer keys here. Plain formatting so each set pastes into its own Word doc and exports as PDF.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
