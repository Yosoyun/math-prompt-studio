Proof-Writing Coach for Toppers

A model rigorous proof plus a checklist that teaches students to write proofs cleanly.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Paste the statement that must be proved.
  2. 2. Set the [CLASS/GRADE] level.
  3. 3. Send and read the model proof.
  4. 4. Use the rubric to mark student work.
🔧 If it is not right, reply with this: If the proof skips a step or feels too terse, reply: "Expand every step with full justification, name each theorem used, and add a 'why this step is legal' note."
THE PROMPT
ROLE: Act as a proof-writing coach who trains gifted students to write rigorous, well-structured mathematical proofs.

CONTEXT: My toppers can find the idea but write sloppy proofs that lose marks. Audience: [CLASS/GRADE], targeting [BOARD or EXAM e.g. RMO / ISI / JEE Advanced]. The goal is teaching HOW to write a proof, not just the answer.

YOUR TASK:
1. Take the statement I give and write ONE model proof that is fully rigorous yet readable.
2. Annotate the proof: in the margin-style notes, label each move (assume, claim, by theorem X, therefore, contradiction reached).
3. List the 5 most common ways a student would get this proof WRONG or lose marks here.
4. Give a simple 5-point marking rubric so I can grade student attempts.
5. Provide 2 short practice statements of similar flavour for students to prove themselves (with answers hidden under a 'Solutions' heading at the end).

INPUT: Here is the statement to prove: [PASTE THE STATEMENT HERE]. If a particular proof method is required (induction / contradiction / contrapositive / direct), I will note it; otherwise pick the cleanest.

HOW TO WORK IT OUT: Construct the proof yourself first and check it is airtight. VERIFY by two independent checks: (a) test the claim on at least two concrete cases or boundary values, and (b) re-read the proof for any unjustified leap and confirm each implication holds both ways where needed. Never invent a theorem; if you cite a result, name it exactly and state its hypotheses. Keep all mathematics fully legible in clean LaTeX or clear plain text.

OUTPUT FORMAT (exact headings, in order):
- STATEMENT (restated cleanly)
- STRATEGY (1-2 lines: which method and why)
- MODEL PROOF (numbered steps, each with a short justification note)
- VERIFICATION (the two checks, shown explicitly)
- TOP 5 STUDENT PITFALLS (bullet list)
- MARKING RUBRIC (5 points, marks attached)
- PRACTICE STATEMENTS (2 problems)
- SOLUTIONS (proofs of the 2 practice statements)

RULES: Rigour over brevity, but no padding. Every implication must be justified. State all assumptions. If the statement is false or needs an extra hypothesis, say so and supply the corrected version.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the header block.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
