Exam-Pattern Mock Section (Timed, With Marking Scheme)

A single timed mock section that matches the real exam pattern, with marking and an answer key.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Open any AI chat.
  2. Type the exam and which section.
  3. Paste the prompt and send.
  4. Give students the set time.
  5. Mark with the scheme and key.
🔧 If it is not right, reply with this: If the marking does not match the real exam, reply: "The marking scheme is wrong for [EXAM] - correct it to the official +/- values for each question type and recompute the maximum marks."
THE PROMPT
ROLE: Act as an official-style mock-test setter who reproduces the exact pattern of a real exam section.

CONTEXT: My students need a realistic timed mock for ONE section of [EXAM e.g. JEE Main Maths / NEET Physics / Board Maths] at [CLASS/GRADE] level. It must match the real pattern: number of questions, question types, marking scheme and time.

YOUR TASK:
1. State the real pattern for this section first (number of questions, types, +/- marking, time) and confirm it matches [EXAM].
2. Build ONE complete mock section that follows that pattern exactly, covering these chapters: [LIST CHAPTERS] with a realistic difficulty spread.
3. Provide a clear marking scheme, the time limit at the top, and an answer key.
4. Add brief solutions for the harder questions.

INPUT: Here is the exam, section and chapters: [PASTE DETAILS HERE]. If a field is blank, assume the standard current pattern for that section and state the assumption in one line at the top. Stay inside the listed chapters.

HOW TO WORK IT OUT: Solve every question internally and VERIFY each answer by two independent checks. Make sure the count of questions, the marking and the maximum marks are internally consistent (the maximum must equal questions times marks per question for each type). If they do not add up, fix it before showing. Never invent formulas or values. Keep maths legible in clean LaTeX or clear plain text.

OUTPUT FORMAT, in this exact order:

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
A. Title block: 'Exam: [EXAM]', 'Section + Time + Maximum Marks', 'Compiled by Indrajeet Yadav'.
B. PATTERN CONFIRMATION: one short block stating the pattern this section follows.
C. SECTION 1 - INSTRUCTIONS: numbered, including the exact marking scheme.
D. SECTION 2 - QUESTIONS: numbered with marks shown and types labelled.
E. SECTION 3 - ANSWER KEY: compact.
F. SECTION 4 - SOLUTIONS for harder questions, with the two checks noted.

RULES: Pattern must match the real exam. Marks must add up to the stated maximum. Every answer survives two checks. No invented values.

At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block.
