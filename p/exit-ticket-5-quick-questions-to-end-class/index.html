Exit Ticket (5 Quick Questions to End Class)

A tiny 5-question exit ticket that checks if today's lesson landed, with instant answer key.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Type today's exact lesson point.
  2. 2. Add class and board.
  3. 3. Send the prompt.
  4. 4. Glance at the answer key.
  5. 5. Print small slips and hand out.
🔧 If it is not right, reply with this: If questions miss today's point, reply: "These don't test what I taught. Focus all 5 questions only on [TODAY'S EXACT SUB-TOPIC] and re-verify."
THE PROMPT
ROLE: Act as a mathematics teacher who writes fast end-of-class exit tickets to check understanding.

CONTEXT: I teach [CLASS/GRADE] under [BOARD or EXAM e.g. CBSE / ICSE / State Board]. In the last 5 minutes of class I give a tiny quiz to see if today's lesson was understood. Today I taught [TODAY'S EXACT SUB-TOPIC].

YOUR TASK:
1. Create exactly 5 short questions, all on [TODAY'S EXACT SUB-TOPIC], answerable in under 5 minutes total.
2. Make Q1-Q2 very easy (recall), Q3-Q4 medium (apply), Q5 a short thinking check.
3. Keep each question to one line where possible; no long word problems.
4. Each question 1 mark; total 5 marks.

INPUT: Here is exactly what I taught today and any example I used: [PASTE TODAY'S LESSON POINT HERE].

HOW TO WORK IT OUT: Solve all 5 internally first. VERIFY each final answer by two independent quick checks. Never invent formulas or values. Keep all maths fully legible in clean LaTeX or clear plain text.

OUTPUT FORMAT (exact headings, in order):
- HEADER LINE: 'Exit Ticket - [Sub-topic]', Class, Total 5 marks, and 'Compiled by Indrajeet Yadav'.
- QUESTIONS: Q1-Q5, numbered.
- ANSWER KEY: the 5 final answers in a separate block.
- TEACHER SIGNAL: one line telling me what a wrong Q5 means students still don't understand.

RULES: Strictly today's sub-topic only. Genuinely quick. Single correct answers. Answer key must match. Very short wording.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
