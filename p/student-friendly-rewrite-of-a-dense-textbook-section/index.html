Student-Friendly Rewrite Of A Dense Textbook Section

A clear, faithful rewrite of a heavy textbook passage with intuition, worked examples and a glossary, while keeping all the rigour.

Works on any free AI  ·  Best tool:  Claude (free) or ChatGPT (free)

✅ How to use this effectively

  1. 1. Paste the original section verbatim.
  2. 2. Name the class and board.
  3. 3. Ask it to keep all rigour intact.
  4. 4. Request added examples and a glossary.
  5. 5. Compare side by side, then export.
🔧 If it is not right, reply with this: If the rewrite oversimplifies and drops a result, reply: 'You omitted [result]. Restore every theorem and condition from the original, just explained more clearly, and add one example showing it.'
THE PROMPT
Act as an expert mathematics editor who rewrites dense, intimidating textbook prose into clear, student-friendly explanations without sacrificing rigour.

CONTEXT: I will paste a dense section from a textbook. Rewrite it for [CLASS/GRADE] students preparing for [BOARD or EXAM]. The rewrite must be faithful: every definition, theorem, condition and result in the original must survive, just explained far more clearly.

YOUR TASK:
1. First, briefly list (internally then for me) the KEY RESULTS the original section actually contains, so nothing is lost.
2. Rewrite the section in plain, encouraging language: short sentences, intuition before symbols, and a worked example for each major idea.
3. Keep every formula, theorem and condition - clearly boxed - and add a one-line 'in plain words' gloss under each.
4. Insert 2-4 NEW worked examples that the original lacked, fully solved.
5. Add MARGIN NOTES style callouts: 'watch out', 'memory hook', 'why this matters'.
6. Provide a GLOSSARY of the technical terms used, each with a one-line meaning.
7. End with a short 'Did I keep everything?' CHECKLIST mapping each original key result to where it now appears.

INPUT: Here is the original dense section: [PASTE THE TEXTBOOK PASSAGE].

HOW TO WORK IT OUT: Before rewriting, extract the full list of results so none is dropped. VERIFY every worked example by two independent methods. Do NOT change the mathematics, weaken any theorem, or invent new results; only the explanation gets clearer. If the original has an error or unstated condition, flag it in a separate 'Editor's Note' rather than silently changing it. Keep all maths in clean ASCII (^, sqrt(), a/b, brackets) and fully legible.

OUTPUT FORMAT (exact order): TITLE BLOCK ('Student-Friendly Rewrite', subtitle naming the section, 'Compiled by Indrajeet Yadav'); 1. Key Results Preserved (checklist of what must survive); 2. The Rewrite (with boxed results, plain-words glosses, callouts); 3. Added Worked Examples; 4. Glossary; 5. Editor's Notes (if any); 6. 'Did I Keep Everything?' Mapping Table.

RULES: Plain ASCII maths, straight quotes, normal hyphens. No loss of rigour, no fabricated results, no silent edits to the original. Faithful first, friendly always.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block.
