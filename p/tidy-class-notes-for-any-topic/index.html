Tidy Class Notes for Any Topic

Well-structured, board-ready class notes for one full topic.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Type your topic and class level.
  2. 2. Paste the whole prompt.
  3. 3. Read the notes top to bottom.
  4. 4. Ask for shorter or longer if needed.
🔧 If it is not right, reply with this: If a section is too thin, reply: "Expand the [section name] section with one more worked example and verify the answer twice."
THE PROMPT
ROLE: Act as an expert mathematics teacher who writes crystal-clear class notes.

CONTEXT: I teach [CLASS/GRADE] students following [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. I want tidy, complete class notes on the topic: [TOPIC NAME]. The notes will be read on the board and copied by students, so they must be clean, ordered, and free of clutter.

YOUR TASK:
1. Write structured class notes covering the whole topic from basics to the level expected for [CLASS/GRADE].
2. Define every term in simple language before using it.
3. List all key formulas with the meaning of each symbol.
4. Give 2 to 3 fully worked examples of increasing difficulty.
5. End with 5 short practice questions (answers at the very bottom only).

INPUT: Here is the topic: [TOPIC NAME]. If I paste any syllabus points or a textbook chapter name, follow them exactly.

HOW TO WORK IT OUT: Solve every example internally first. Then VERIFY each final answer by at least two independent checks (for example, re-substitution and an alternative method). Never invent a formula or a value; if something is outside the standard syllabus, say so plainly. Keep all mathematics fully legible using clean LaTeX or clear plain text with one step per line. Do not skip algebra steps.

OUTPUT FORMAT (use these exact headings, in this order):
1. Topic Overview (3 to 4 lines)
2. Key Definitions
3. Formulas (with symbol meanings)
4. Core Concepts (short numbered points)
5. Worked Examples (Example 1, Example 2, Example 3)
6. Common Mistakes to Avoid (4 bullets)
7. Practice Questions (Q1 to Q5)
8. Answer Key (only at the very end)

RULES: Use plain, exam-correct language. No filler, no repetition. Every formula must be standard and correct. Keep each worked example to one clear method. If anything is ambiguous, state your assumption in one line before proceeding.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.

SIGNATURE: End your ENTIRE response with exactly this footer block (keep the lines and symbols):
