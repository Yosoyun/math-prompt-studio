Speed Tricks & Shortcuts for One Topic (Solve Faster)

A list of fast methods and shortcuts for a topic, each shown beside the slow method with timing.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Open any AI chat.
  2. Enter the exact topic.
  3. Paste the prompt and send.
  4. Test each trick on its worked example.
  5. Teach the trick alongside the standard method.
  6. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).

THE PROMPT
ROLE: Act as a master exam-strategy coach famous for legitimate, mathematically valid speed shortcuts.

CONTEXT: My [EXAM e.g. JEE Main / NEET] students lose time on [TOPIC e.g. Quadratic Roots / Permutations / Ratio-Proportion]. I want shortcuts that are genuinely valid (not memorised hacks that break) so they can solve under time pressure.

YOUR TASK:
1. Give EXACTLY [NUMBER e.g. 7] speed techniques for [TOPIC] that save real time in the exam.
2. For each trick: state the trick, when it applies, when it FAILS (its limitation), and a worked example solving the same problem the long way AND the fast way.
3. Show the approximate time saved per trick.
4. End with a 1-minute drill of 5 quick questions to practise the tricks, with answers.

INPUT: Here is the topic and the kind of questions students see: [PASTE DETAILS HERE]. If number is blank, use 7. Stay inside this topic.

HOW TO WORK IT OUT: For every trick, prove it works by solving the example BOTH ways and confirming the answers match; that is your verification. Explicitly test one case where the trick could fail to define its limitation. Never present a shortcut as universal if it has exceptions. Never invent formulas or values. Keep maths legible in clean LaTeX or clear plain text.

OUTPUT FORMAT, in this exact order:

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
A. Title block: 'Topic: [TOPIC]', 'Speed Tricks for [EXAM]', 'Compiled by Indrajeet Yadav'.
B. SECTION 1 - TRICKS: for each trick, the four parts (What / When it applies / When it fails / Worked example long-way vs fast-way) and time saved.
C. SECTION 2 - PROOF CHECK: one line per trick confirming both methods gave the same answer.
D. SECTION 3 - 1-MINUTE DRILL: 5 quick questions with an answer key.

RULES: Every trick must be mathematically valid with stated limits. Both-method check is mandatory. No invented values. No 'works always' claim unless it truly does.

At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block.
