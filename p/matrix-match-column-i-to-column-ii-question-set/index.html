Matrix-Match (Column I to Column II) Question Set

Matrix-match questions with correct A-r type matchings and full justification for each link.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Open any AI chat.
  2. Type the topic for the match set.
  3. Paste the prompt and send.
  4. Hand out the two-column questions.
  5. Mark using the matching key.
  6. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If a Column I entry has no valid match, reply: "Entry (B) has no correct match in Column II - fix the columns so every Column I entry matches at least one Column II option, then re-verify all matchings."
THE PROMPT
ROLE: Act as a JEE Advanced matrix-match (matching-type) question designer and verifier.

CONTEXT: My [EXAM e.g. JEE Advanced] students need matrix-match practice on [TOPIC] at [CLASS/GRADE] level. In these, Column I has entries (A,B,C,D) to be matched to Column II options (p,q,r,s,t); an entry may match more than one option.

YOUR TASK:
1. Create EXACTLY [NUMBER e.g. 4] matrix-match questions on [TOPIC].
2. Each has Column I with 4 entries (A-D) and Column II with options (p,q,r,s,t). Allow multiple correct matches per entry where natural.
3. Provide the correct matching for every entry and a one-line justification for each match.
4. Keep the entries conceptually varied so the question tests breadth.

INPUT: Here is the topic and sub-areas: [PASTE DETAILS HERE]. If number is blank, use 4. Stay inside this topic.

HOW TO WORK IT OUT: Work out every entry's correct option(s) internally, then VERIFY each match by an independent check (re-derive the result, or test a value). Confirm every Column I entry has at least one valid match and no match is accidentally ambiguous beyond what you intend. Never invent formulas or values. Keep maths legible in clean LaTeX or clear plain text.

OUTPUT FORMAT, in this exact order:

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
A. Title block: 'Topic: [TOPIC]', 'Type: Matrix-Match', 'Exam: [EXAM]', 'Compiled by Indrajeet Yadav'.
B. SECTION 1 - QUESTIONS: each with Column I (A-D) and Column II (p-t) clearly laid out.
C. SECTION 2 - MATCHING KEY: e.g. 'A-(p,r), B-(q), C-(s,t), D-(p)'.
D. SECTION 3 - JUSTIFICATIONS: one short verified line per match explaining why it holds.

RULES: Every entry matches at least one option. Matches verified independently. No invented values. Conceptually varied entries.

At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block.
