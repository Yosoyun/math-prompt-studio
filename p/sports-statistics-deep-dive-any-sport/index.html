Sports Statistics Deep Dive (Any Sport)

Stats problems from football, kabaddi, athletics or any sport students follow.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Name the sport your class loves.
  2. 2. Add the topic and class.
  3. 3. Send and read the stats problems.
  4. 4. Ask for a team-comparison question.
  5. 5. Mark with the answer key.
🔧 If it is not right, reply with this: If an average or percentage is wrong, reply: 'Recompute every statistic step by step and verify each answer a second way.'
THE PROMPT
ROLE: Act as a maths teacher who teaches statistics through sport.
CONTEXT: My students are in [CLASS/GRADE] under [BOARD or EXAM] and follow [SPORT e.g. football / kabaddi / badminton / athletics]. I want to teach [TOPIC e.g. mean / median / mode / percentages / data handling] using realistic match and player statistics.
YOUR TASK:
1. Build a small realistic data set (a short table of players or matches with stats).
2. Create 8 problems: find the average, compare two players, find the best performer, percentage success rate, spot the outlier, predict from a trend.
3. Easy to hard.
4. Full solution for each.
INPUT: Here is the sport and topic: [PASTE HERE]. If I give my own data table, use it exactly.
HOW TO WORK IT OUT: Solve internally first. VERIFY each statistic by two independent checks (recompute, and confirm the answer is reasonable for the data). Use correct statistical definitions; never invent a formula. If you create the data, keep all numbers realistic for the sport. Keep working legible.
OUTPUT FORMAT:
- 'The Data' (a clean table).
- 'Stats Problems' numbered 1 to 8 with marks.
- 'Solutions' (full working, each ending with 'Check:').
- 'Selector's Insight' (2 lines on how real coaches use these numbers).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
RULES: Realistic data, correct stats definitions, simple language, nothing above class level.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
