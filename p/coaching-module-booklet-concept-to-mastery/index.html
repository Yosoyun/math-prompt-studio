Coaching Module Booklet (Concept To Mastery)

A self-contained coaching-style module: theory recap, technique boxes, model solved problems, a graded DPP and a short test with full solutions.

Works on any free AI  ·  Best tool:  Claude (free) or ChatGPT (free)

✅ How to use this effectively

  1. 1. Set topic, exam and difficulty.
  2. 2. Say how many DPP and test questions.
  3. 3. Ask for theory + techniques first.
  4. 4. Then ask for DPP, then the test.
  5. 5. Paste into Word and print.
🔧 If it is not right, reply with this: If problems feel too easy or repetitive, reply: 'Raise the difficulty to top-rank level, remove any near-duplicate questions, and add 3 multi-concept problems.'
THE PROMPT
Act as a master coaching-institute faculty member designing a printed self-study module for serious aspirants.

CONTEXT: The module is for [CLASS/GRADE] students preparing for [EXAM]. The topic is [TOPIC]. Target standard is [easy / standard / top-rank]. The booklet must take a student from concept to exam-mastery without any other resource.

YOUR TASK:
1. Write a CONCEPT RECAP (the 20% of theory that drives 80% of questions), with boxed key results.
2. Add a TECHNIQUES section: 4-6 named problem-solving techniques/shortcuts, each with when-to-use, a mini-example, and a caution.
3. Provide 8 MODEL PROBLEMS (graded), each fully solved with the chosen technique flagged and a one-line takeaway.
4. Build a DAILY PRACTICE PROBLEM set (DPP) of [N] questions with a clean answer key (final answers only).
5. Build a TIMED MINI-TEST of [M] questions with marks, suggested time, and FULL worked solutions.
6. Add a 'TRAPS ON THIS TOPIC' callout listing the 5 most common ways students lose marks here.

INPUT: Subtopics to include: [PASTE SUBTOPICS]. Preferred N (DPP) = [N], M (test) = [M]. (If blank, use N=15, M=8.)

HOW TO WORK IT OUT: Solve everything internally before writing. VERIFY each answer by two independent methods and flag any question whose answer you are not fully certain of. Never fabricate formulas or values; keep conditions explicit. Use clean linear ASCII maths (^, sqrt(), a/b, brackets). Ensure no two problems are near-duplicates and that difficulty genuinely increases.

OUTPUT FORMAT (exact order): HEADER ('[TOPIC] - Coaching Module', subtitle 'Compiled by Indrajeet Yadav'); A. Concept Recap; B. Techniques (T1...T6); C. Model Problems (with technique tags); D. DPP (questions then Answer Key); E. Mini-Test (questions, then Full Solutions); F. Traps On This Topic; G. One-Page Cheat Strip (formulas + technique triggers).

RULES: Plain ASCII only, straight quotes, normal hyphens. No padding, no invented references. Mark each problem's difficulty (E/S/C). If output is long, deliver in labelled parts and prompt me to say 'continue'.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the header block.
