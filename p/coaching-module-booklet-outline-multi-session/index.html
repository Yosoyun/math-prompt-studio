Coaching Module Booklet Outline (Multi-Session)

A session-wise module plan for a coaching booklet, ready to fill in.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Enter topic, class, board and session count.
  2. 2. State total teaching hours if known.
  3. 3. Send the prompt.
  4. 4. Adjust pacing per session as needed.
🔧 If it is not right, reply with this: If a session is overloaded, reply: "Session 3 has too much; split it into two sessions and rebalance the homework load."
THE PROMPT
ROLE: Act as a coaching-institute academic head designing a module booklet.

CONTEXT: I am building a coaching module booklet for [CLASS/GRADE] students of [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET] on [TOPIC OR UNIT]. It will run over [NUMBER e.g. 6] sessions of about [DURATION e.g. 90 minutes] each. I need a clear session-wise plan before writing the content.

YOUR TASK:
1. Design a session-by-session plan for the whole module.
2. For each session, give: objective, concepts to teach, board-work examples count, in-class practice count, and homework count.
3. Sequence sessions so each builds on the previous one.
4. Add a short diagnostic test at the start and a module test at the end.
5. Suggest difficulty progression across sessions.
6. Estimate time split within each session (teach / practise / review).

INPUT: Here is the scope and any constraints (hours, pace, weak areas): [PASTE HERE].

HOW TO WORK IT OUT: Plan the dependency order and pacing internally first. VERIFY two ways: (a) confirm each session's prerequisites are taught in an earlier session, and (b) confirm the total content fits the stated session count and durations without overload. Do not invent non-syllabus content. Keep any sample notation legible in clean LaTeX or clear plain text.

OUTPUT FORMAT (exact order):
1. Module Title and Overview
2. Diagnostic Test (brief description)
3. Session-by-Session Plan (per session: objective, concepts, examples, practice, homework, time split)
4. Difficulty Progression Note
5. End-of-Module Test (brief description)

RULES: Logical session order. Realistic time and load per session. No overloaded sessions. Nothing outside the stated scope.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the header block at the top.
