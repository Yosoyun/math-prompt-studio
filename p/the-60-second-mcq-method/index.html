The 60-Second MCQ Method

The fastest valid trick to crack an MCQ, plus the full method to confirm it.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Paste the prompt into any AI chat.
  2. 2. Fill class and exam brackets.
  3. 3. Paste the question AND all four options.
  4. 4. Send and learn the fast trick.
  5. 5. Teach it as a time-saver for exams.
🔧 If it is not right, reply with this: If the shortcut is risky, reply: 'Show one option-elimination trick and confirm it cannot pick a wrong option by mistake.'
THE PROMPT
ROLE: Act as a competitive-exam coach who finds the fastest valid way to crack a multiple-choice question.

CONTEXT: For a [CLASS/GRADE] student in [BOARD or EXAM e.g. JEE / NEET / State CET / CBSE MCQ]. In MCQs, speed wins. The teacher wants the quickest legitimate method PLUS a full method to confirm it is safe to teach.

YOUR TASK:
1. Identify the correct option to the MCQ below.
2. FAST METHOD: the quickest valid route to the answer in under a minute - this may use plugging in the options, special values, dimensional/unit reasoning, elimination, estimation, or symmetry. Numbered, very short.
3. FULL METHOD: the complete proper solution, numbered steps, to confirm the fast method is correct.
4. Give one line on the time-saving trick and one trap to avoid.

INPUT: Here is the question and ALL options: [PASTE THE QUESTION AND OPTIONS A, B, C, D HERE]

HOW TO WORK IT OUT: Solve privately and lock the correct option. The fast method and the full method must agree on the same option. Verify with two independent checks. Never invent formulas or values; name any used. Make the fast method genuinely fast but valid - not a blind guess. Keep all maths legible in clean LaTeX or plain text.

OUTPUT FORMAT (exact order):
CORRECT OPTION (e.g. Answer = C)
FAST METHOD (under 60 seconds; very short numbered steps)
FULL METHOD (complete numbered steps confirming the option)
TRICK and TRAP (one line each)
VERIFICATION (two independent checks)

RULES: Fast and full methods must select the identical option. The fast method must be reliable, not a gamble. If options are missing, ask for them. Use exact values where possible.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
