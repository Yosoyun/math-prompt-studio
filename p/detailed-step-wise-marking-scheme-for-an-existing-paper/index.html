Detailed Step-Wise Marking Scheme for an Existing Paper

A step-wise marking scheme that splits each answer's marks across the working steps.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Copy your existing paper text.
  2. 2. Paste it where the prompt says.
  3. 3. Add the prompt below it.
  4. 4. Send and read the per-step marks.
  5. 5. Confirm each question's steps sum correctly.
🔧 If it is not right, reply with this: If a question's step marks do not add up, reply: 'The step marks for question [number] do not sum to its total. Re-split the marks so they add up exactly.'
THE PROMPT
ROLE: Act as a senior mathematics examiner who writes official-style step-wise marking schemes.

CONTEXT: I already have a question paper. I need a DETAILED MARKING SCHEME for it, for Class [CLASS/GRADE], [BOARD or EXAM e.g. CBSE / ICSE / State Board], so different teachers grade it identically and fairly.

YOUR TASK:
1. For every question, give the full correct solution.
2. Break each question's marks across the working steps (for example: correct setup 1 mark, correct substitution 1 mark, correct final answer 1 mark).
3. List acceptable alternative methods that should also earn full marks.
4. Note common errors and how many marks to deduct for each.

INPUT: Here is the question paper: [PASTE THE FULL PAPER HERE, including marks per question]. If marks per question are missing, assume a sensible split and state it.

HOW TO WORK IT OUT: Solve every question internally first, then verify each final answer twice by independent methods. Confirm the step marks for each question add up exactly to that question's total. Never invent formulas or values. Keep all mathematics legible in clean LaTeX or plain text.

OUTPUT FORMAT, in this order:
1. HEADER: 'Marking Scheme - Mathematics', class, board, total marks, 'Compiled by Indrajeet Yadav'.
2. For EACH question, in order: full solution, then a STEP-MARKS table (step described, marks for that step), then 'Acceptable alternatives', then 'Common errors and deductions'.
3. A final line confirming the scheme totals match the paper total.

RULES: Step marks per question must sum exactly to that question's marks. Be fair and explicit. Cover every question.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the header.
