THE PROMPT
You are a senior Olympiad mathematics editor and professional LaTeX typesetter. TASK: build a REUSABLE LaTeX exam/worksheet TEMPLATE that a maths teacher can edit again and again, and prove it works by compiling ONE sample PDF from it with pdflatex. Deliver the .tex template file AND the compiled sample PDF.

PARAMETERS (I fill these):
- Document kind: [KIND] (exam paper / worksheet / DPP / assignment)
- Institute / class line: [INSTITUTE / CLASS]
- Default TOTAL MARKS and DURATION: [TOTAL MARKS] / [DURATION]
- Section layout: [SECTIONS] (e.g. Section A objective, Section B subjective)
- Marks column: [MARKS COLUMN] (right-aligned per question yes/no)
- Sample TOPIC for the demo paper: [TOPIC]
- Number of demo questions: [NUMBER]
- Footer text: Indrajeet Yadav

PHASE 1 - DESIGN THE TEMPLATE STRUCTURE. Decide a clean, professional skeleton driven by easy-to-edit macros at the top of the file: \newcommand for the paper title, [INSTITUTE / CLASS], [TOTAL MARKS], [DURATION] and the instruction block, so the teacher changes one line, not the body. Provide a simple custom \question macro (and an optional marks argument) plus an answer-key environment, so adding/removing questions is trivial. Lay out [SECTIONS] with clear headings and automatic question numbering via enumitem.

PHASE 2 - WRITE A DEMO PAPER. Inside the template, fill in [NUMBER] genuinely [TOPIC] questions at serious level (RMO/INMO/JEE-Advanced/Putnam calibre) so the sample looks real - not 'Lorem ipsum' placeholders. These demo questions exist only to exercise every feature of the template (display math, cases, a short table, multi-part question, a proof-style item).

PHASE 3 - SOLVE AND VERIFY THE DEMO. Independently solve every demo question from scratch; never trust a guessed answer. Use EXACT values. For any find-all item prove necessity AND sufficiency; for proofs, complete rigorous arguments with no circular reasoning and no 'clearly'; check all cases and equality conditions; do not cite any theorem you have not stated. Put these verified answers into the answer-key environment so the sample's key is correct.

PHASE 4 - TYPESET, COMMENT AND COMPILE.
LaTeX rules:
- documentclass[11pt]{article}; default Computer Modern (no font packages).
- Permitted packages ONLY: amsmath, amssymb, enumitem, geometry, array, fancyhdr. Use \usepackage[margin=1in]{geometry}.
- Footer via fancyhdr, no header/footer rules:
```
\pagestyle{fancy}
\fancyhf{}
\fancyfoot[C]{Indrajeet Yadav}
\renewcommand{\headrulewidth}{0pt}
\renewcommand{\footrulewidth}{0pt}
```
- Use \dfrac, \left/\right, \lfloor/\rfloor, \lceil/\rceil, \binom, \pmod, \mathbb; display math via \[ ... \] with aligned/cases; never Unicode math when a LaTeX command exists.
- IMPORTANT: scatter clear % EDIT HERE comments through the template marking exactly where the teacher changes the title, marks, sections and questions. The file must be self-documenting.

DESIGN RULES: strict black-and-white minimalist. No colours, boxes, shading, borders, icons, logos, watermarks, decorations, difficulty stars or banners. Clean academic typography only.

DELIVERABLES (think of the sample as the two-PDF idea folded into one demo): the .tex TEMPLATE must itself be able to produce BOTH a question view and a solutions/answer-key view via a single toggle macro at the top, e.g. \newcommand{\showsolutions}{false} that, when set true, prints the answer-key table and solutions S1, S2, ...; when false, prints questions only with NO answer leakage. Demonstrate the false (questions-only) state in the compiled sample.
- Template file: Exam_Template_Indrajeet_Yadav.tex
- Compiled sample PDF: Exam_Template_Sample_Indrajeet_Yadav.pdf

COMPILE STEP: compile the sample with pdflatex, read the log, fix EVERY error, warning, overfull/underfull line, broken formula and bad page break; recompile until clean; flip \showsolutions to true once internally to confirm the solutions branch also compiles, then set it back to false for the delivered sample; visually check every page.

HONESTY: never fabricate theorems, statistics or results. If a parameter is missing or contradictory, ask me before guessing.

FINAL DELIVERY: return ONLY the downloadable .tex file link and the sample PDF link plus ONE sentence confirming the sample compiled cleanly with pdflatex (and that the solutions toggle also compiles). Do NOT paste the template source, questions or solutions into the chat.
