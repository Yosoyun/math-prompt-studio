Whole-Syllabus Overview Map

A bird's-eye map of the entire year's syllabus, unit by unit.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Type the class and board or exam.
  3. 3. Paste the prompt below.
  4. 4. Send and review every unit.
  5. 5. Ask it to zoom into any one unit.
🔧 If it is not right, reply with this: If a chapter is missing, reply: "Add the missing chapters for [class] [board] and re-check the unit grouping." If it overreaches, reply: "Remove anything not in the official [board] syllabus for this class."
THE PROMPT
ROLE: Act as a senior mathematics teacher and curriculum planner who maps an entire year's syllabus at a glance.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. I want a WHOLE-SYLLABUS OVERVIEW MAP (text form) of the complete year - a bird's-eye view organised by units and chapters - so I can plan teaching order and show students the big picture on day one. No image needed.

YOUR TASK:
1. Put the subject and class as the centre.
2. Make one main branch per UNIT of the official syllabus.
3. Under each unit list its chapters.
4. Under each chapter give a 3 to 6 word summary of its core idea (not full detail).
5. Add a SUGGESTED TEACHING ORDER and mark which units are heaviest (most exam weight / most teaching time).

INPUT: Here is the class, board and any syllabus notes I have: [PASTE CLASS, BOARD AND ANY SYLLABUS DETAILS HERE]. If I give only class and board, use the standard official syllabus structure for [CLASS/GRADE].

HOW TO WORK IT OUT: First recall the official unit and chapter structure, then organise it cleanly. VERIFY the chapter list twice against the standard [BOARD] syllabus - do not add chapters that are not part of it and do not drop required ones. If you are unsure a chapter belongs, mark it "(verify against official syllabus)". Keep summaries short and accurate. Use clean plain text.

OUTPUT FORMAT (exactly this):
SUBJECT: Mathematics - <class> (<board/exam>)
UNIT 1 - <name>
   Chapter: <name> - <core idea in a few words>
   Chapter: <name> - <core idea>
UNIT 2 - <name>
   Chapter: <name> - <core idea>
(continue for all units)
SUGGESTED TEACHING ORDER: list the units in a sensible sequence.
HEAVIEST UNITS: name the 2 or 3 units needing the most time and why.

RULES: Stick strictly to the official syllabus. Keep summaries short. Be complete but not detailed - this is an overview.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.

SIGNATURE: End your ENTIRE response with exactly this footer block (keep the lines and symbols):
