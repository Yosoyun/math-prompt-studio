True or False with Justification

A set of True/False statements that force real understanding, each with the correct verdict and a one-line reason.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Fill class, board and chapter.
  2. 2. Choose how many statements you want.
  3. 3. Send the prompt.
  4. 4. Confirm the mix of true and false.
  5. 5. Read each one-line justification.
🔧 If it is not right, reply with this: If a statement is ambiguous, reply: 'Statement 5 could be read two ways - reword it so it is clearly either true or false, then re-verify the verdict.'
THE PROMPT
ROLE: Act as a mathematics teacher who designs True/False statements that test conceptual understanding, not memory.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. The chapter is [CHAPTER NAME]. I want [NUMBER, e.g. 12] True/False statements.

YOUR TASK:
1. Write [NUMBER] statements about the named chapter, each clearly TRUE or clearly FALSE.
2. Make roughly half true and half false, in random order.
3. Design the FALSE statements around real misconceptions students hold (for example a sign rule, a special case, a wrong assumption that 'always' or 'never' holds).
4. Keep each statement to one clean sentence with clearly written maths.

INPUT: Here is the sub-topic or any specific misconception I want to target: [PASTE TOPIC OR LEAVE BLANK].

HOW TO WORK IT OUT: Decide each verdict by solving or testing it, including edge cases. VERIFY by a second method or a counter-example, especially for any statement using 'always', 'never', or 'only'. Do not invent rules. Keep notation legible.

OUTPUT FORMAT:
SECTION 1 - STATEMENTS: A numbered list, statements only, no answers.
SECTION 2 - ANSWER KEY: Table with columns: No | True or False | One-line reason | If false, the correct version or counter-example.

RULES: No ambiguous wording. No statement that depends on opinion. Every false statement must be defensibly false with a reason. Difficulty suited to the class above.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
