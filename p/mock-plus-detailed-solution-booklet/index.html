Mock plus Detailed Solution Booklet

A clean question paper, then a teaching-style solution booklet with method notes.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Enter class, board, chapters, marks.
  2. 2. Send the prompt.
  3. 3. Read the paper, then the booklet.
  4. 4. Ask to expand any solution further.
  5. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If a solution skips reasoning, reply: "For question 4, add the 'why this method' note and a common mistake students make here."
THE PROMPT
ROLE: Act as a master teacher who writes detailed solution booklets, not just answer keys.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / JEE]. I want a mock paper plus a rich solution BOOKLET that students can learn from on their own, because it explains the thinking behind each step.

YOUR TASK:
1. Build a full mock for these chapters: [LIST CHAPTERS]. Total marks: [TOTAL MARKS]. Time: [TIME].
2. Then write a DETAILED SOLUTION BOOKLET. For each question include: the full worked steps, a short 'Why this method' note, and a 'Common mistake' warning.
3. Where useful, give an alternative shorter method.

INPUT: Here are the chapters and marks: [PASTE DETAILS HERE].

HOW TO WORK IT OUT: Solve every question internally first. VERIFY each final answer two independent ways. Never invent formulas or values. Make every step legible in clean plain text or LaTeX; do not skip a single algebra step. Keep the 'why' notes short and genuinely useful.

OUTPUT FORMAT (exact order):
PAPER: header, sections, numbered questions, marks shown.
--- SOLUTION BOOKLET ---
For each question, a block with these sub-labels:
  Steps: full working.
  Answer: final result labelled.
  Why this method: 1-2 lines.
  Common mistake: 1 line.
  Faster way (optional): 1-2 lines if a neat shortcut exists.
VERIFICATION NOTE: confirm every answer was double-checked.

RULES: No skipped steps. Marks total [TOTAL MARKS]. Original questions. Difficulty true to [BOARD or EXAM]. Notes must be honest and helpful, not filler.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.

SIGNATURE: End your ENTIRE response with exactly this footer block (keep the lines and symbols):
