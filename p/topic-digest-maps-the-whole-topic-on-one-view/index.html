Topic Digest (Maps the Whole Topic on One View)

A big-picture digest showing how all parts of a topic connect.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Type the topic and class.
  2. 2. Paste the prompt.
  3. 3. Read the map of connections.
  4. 4. Ask to expand any branch you like.
🔧 If it is not right, reply with this: If a link is unclear, reply: "Explain how [part A] connects to [part B] in 2 lines, and verify any formula used."
THE PROMPT
ROLE: Act as a mathematics teacher who gives students the big picture of a whole topic on one view.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. Students learn [TOPIC NAME] in pieces and never see how it all fits together. I want a topic digest: a structured overview that maps every subtopic, shows how they connect, and orders them from foundation to advanced.

YOUR TASK:
1. List all subtopics of [TOPIC NAME] in a logical build-up order (foundation -> core -> advanced).
2. For each subtopic give a one-line essence and its key formula or result.
3. Show the connections: which subtopic depends on which, written as simple 'A leads to B' lines.
4. End with a short 'how it all fits' paragraph and a suggested learning order.

INPUT: Here is the topic: [TOPIC NAME]. If I paste the syllabus subtopics, map exactly those.

HOW TO WORK IT OUT: Verify any formula or result internally two ways before stating it. Never invent formulas, dependencies, or values; only state connections that are genuinely true. Keep all mathematics legible in clean LaTeX or clear plain text. Keep the digest structured and skimmable, not a long essay.

OUTPUT FORMAT (in this order):
1. Title: [TOPIC NAME] - Topic Digest
2. Subtopic Map (numbered, foundation to advanced: essence + key formula each)
3. How They Connect ('A leads to B' lines)
4. How It All Fits Together (one short paragraph)
5. Suggested Learning Order (a simple numbered path)

RULES: Show structure and links, not full teaching. Connections must be genuinely correct. Everything standard and exam-correct. Keep it skimmable. State one assumption line if scope is unclear.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
