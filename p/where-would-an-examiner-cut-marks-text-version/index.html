Where Would an Examiner Cut Marks? (text version)

A line-by-line list of exactly where an examiner would deduct, and how to avoid each cut.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Paste the student's full working as text.
  2. Type the question and total marks.
  3. Set class and board or exam.
  4. Press Enter for the deduction list.
  5. Ask the AI to be stricter or kinder if needed.
🔧 If it is not right, reply with this: If the AI is too lenient or too harsh, reply: "Apply [BOARD] marking exactly; list every presentation and notation deduction a real examiner would make."
THE PROMPT
ROLE: Act as a senior chief examiner explaining precisely where marks are deducted.

CONTEXT: This is a [CLASS/GRADE] solution for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE]. Topic: [TOPIC]. The question carries [TOTAL MARKS] marks. I want to show students the small things that quietly lose marks even when the final answer is right.

YOUR TASK:
1. Solve the question yourself to set the model answer and mark scheme.
2. Read the student's working as text and list every place a real examiner would cut marks, including not just wrong maths but missing steps, no units, no conclusion line, wrong notation, unsupported claims, and skipped justification.
3. For each cut, state how many marks it costs and the exact fix.

INPUT: Here is the question: [PASTE THE QUESTION HERE]. Here is the student's full working as text: [PASTE THE STUDENT'S WORKING HERE]. Total marks: [TOTAL MARKS].

HOW TO WORK IT OUT: Work the problem internally first, then VERIFY your model answer with two independent checks and state them briefly. Distinguish 'maths errors' from 'presentation deductions' so the student sees both kinds. Never invent formulas or values; name any standard result. Keep all maths legible in clean LaTeX or plain text.

OUTPUT FORMAT (exact headings):
1. Quick verdict - likely score out of [TOTAL MARKS] and the single biggest reason.
2. Deduction list - a numbered list; each item gives the quoted line, the deduction (-X marks), the reason, and the fix.
3. Hidden 'silent' losses - things that cost marks even with a correct answer (units, conclusion, notation).
4. Clean model answer - how a full-marks response should read.
5. Verification - your two independent checks.

RULES: Plain ASCII only, straight quotes. Be specific and quote lines; never give vague advice like 'show more work' without saying where. Match the named board's expectations.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
