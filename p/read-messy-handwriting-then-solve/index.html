Read Messy Handwriting, Then Solve

A clean typed version of the messy question plus a full, verified solution.

Attach a photo of the question first  ·  Best tool:  ChatGPT or Gemini (image)

✅ How to use this effectively

  1. 1. Photograph the handwritten question as clearly as possible.
  2. 2. Attach the photo to the chat.
  3. 3. Paste this prompt and send.
  4. 4. Check the clean typed version it produced.
  5. 5. Correct any misread symbol, then ask it to re-solve.
🔧 If it is not right, reply with this: If it misreads a symbol, reply: "The symbol on line [X] is actually [+ / x / square root / theta]. Use that, re-type the question, and solve again."
THE PROMPT
ROLE: Act as an expert at reading difficult or messy mathematics handwriting and then solving the problem correctly.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET], topic [TOPIC]. The handwriting in my photo is messy, so I need you to first read it carefully, then solve it.

YOUR TASK:
1. Carefully decode the handwriting in the attached photo, character by character.
2. Produce a CLEAN typed version of the exact question.
3. List any symbols you were unsure about and the assumption you made for each.
4. Solve the question fully once the wording is settled.

INPUT: I have attached a photo of a handwritten question. Watch carefully for easily confused marks: 1 vs 7, x vs multiplication, 0 vs 6, the square root sign, fraction bars, powers, and Greek letters like theta or pi. If a mark is genuinely unreadable, mark it as [unclear] and state your best guess.

HOW TO WORK IT OUT: After decoding, solve the problem silently. Then VERIFY the final answer with TWO independent checks (for example substitute back, and re-derive a second way). If the checks disagree, recheck whether you misread the handwriting before recomputing. Never invent formulas or values; name standard results. Keep all mathematics fully legible in clean LaTeX or clear plain text, one step per line.

OUTPUT FORMAT (use these exact headings):
1. Clean typed question - the decoded question in tidy text.
2. Symbols I was unsure about - a short list, each with my assumption (write "None" if all clear).
3. Given and To find.
4. Step-by-step solution - numbered, one idea per line, short reasons.
5. Final answer - clearly stated.
6. Verification - Check 1 and Check 2, each in one or two lines.

RULES: Do not solve until you have shown me the clean typed question. If your reading changes the meaning, flag it clearly. Use plain ASCII only with straight quotes. Be exact unless an estimate is asked for.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block at the top.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
