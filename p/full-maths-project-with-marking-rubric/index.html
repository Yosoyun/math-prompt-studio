Full Maths Project with Marking Rubric

A complete student project brief plus a fair, ready-to-use marking rubric.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Type your class, board and topic in the brackets.
  2. 2. Paste the whole prompt into any AI chat.
  3. 3. Read the project brief and the rubric.
  4. 4. Copy both into a Word or Google doc.
  5. 5. Share with students; keep the rubric to grade.
🔧 If it is not right, reply with this: If the project is too hard or too easy, reply: "Adjust the difficulty for an average student in this class, keep the same sections, and make the rubric add up to exactly 20 marks."
THE PROMPT
ROLE: Act as an experienced school mathematics teacher and curriculum designer who writes excellent hands-on projects.

CONTEXT: I teach [CLASS/GRADE] following [BOARD or EXAM e.g. CBSE / ICSE / State Board]. I need one complete mathematics PROJECT on the topic [TOPIC e.g. statistics / surface area and volume / coordinate geometry] that students can do in groups of [GROUP SIZE e.g. 3-4] over [TIME e.g. one week]. My students are bright but need very clear, step-by-step instructions.

YOUR TASK:
1. Design one engaging, real-life project on the topic above.
2. Write a student-facing brief with a clear goal and concrete steps.
3. List exactly what materials or data students need.
4. Create a fair marking rubric totalling 20 marks.
5. Add 3 differentiation tips (one easier version, one harder extension, one support idea).

INPUT: Here is my topic and any extra detail: [PASTE TOPIC AND ANY NOTES HERE]. If I leave this blank, choose a strong default for the class and board above.

HOW TO WORK IT OUT: First plan the project internally and make sure every step is doable with ordinary school materials. Check that the mathematics is correct and matches the stated class and board. Verify the rubric marks add up to exactly 20 by listing each criterion's marks and showing the total. Never invent formulas; use only standard, correct ones. Keep all maths in clean, legible plain text or simple LaTeX.

OUTPUT FORMAT (use these exact headings in this order):
1. PROJECT TITLE
2. LEARNING OBJECTIVES (3-4 bullets)
3. MATERIALS / DATA NEEDED
4. STUDENT INSTRUCTIONS (numbered steps, written to the student)
5. SAMPLE MATHEMATICS (one short worked example showing the maths involved)
6. MARKING RUBRIC (a table: Criterion | What earns full marks | Marks; final row Total = 20)
7. DIFFERENTIATION (Easier / Harder / Support)
8. TEACHER TIPS (timing and common mistakes)

RULES: Be specific and classroom-ready. No vague phrases like 'students explore'; say exactly what to do. Keep instructions short and friendly. Make sure the project is original and safe.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
