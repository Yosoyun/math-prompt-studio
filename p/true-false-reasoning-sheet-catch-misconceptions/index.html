True/False Reasoning Sheet (Catch Misconceptions)

20 true/false statements that target common errors, each with a one-line justification key.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Name the topic to probe.
  2. 2. Add class and board.
  3. 3. Send the prompt.
  4. 4. Read the reasoning key.
  5. 5. Print the reasoning sheet.
🔧 If it is not right, reply with this: If a statement is ambiguous, reply: "Statement 12 could be read two ways. Reword it so it is clearly true or clearly false, and update the justification."
THE PROMPT
ROLE: Act as a mathematics teacher who designs true/false reasoning sheets that expose misconceptions.

CONTEXT: I teach [CLASS/GRADE] under [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE]. Students often 'feel' an answer is right without reasoning. I want 20 true/false statements on [TOPIC], deliberately including the common wrong beliefs students hold, so they must think and justify.

YOUR TASK:
1. Write 20 statements about [TOPIC]; aim for a roughly even mix of true and false (around 10 each), in random order.
2. Make several FALSE statements reflect real student misconceptions (the kinds of errors they actually make), not silly obvious mistakes.
3. Require students to write a one-line justification or a counter-example for each, so guessing is not enough.
4. Mark each statement out of 2 (1 for correct T/F, 1 for correct justification). Show the total.

INPUT: Here is the topic and the misconceptions I most want to target: [PASTE TOPIC AND MISCONCEPTIONS HERE].

HOW TO WORK IT OUT: Decide the truth of each statement internally with care. VERIFY each verdict two independent ways (for example, a proof for true, a counter-example for false). Be extremely careful: never label a true statement false or vice versa. Never invent formulas. Keep all maths fully legible in clean LaTeX or clear plain text.

OUTPUT FORMAT (exact headings, in order):
- TITLE BLOCK: 'True or False - Justify', Topic, Class, Total Marks, and 'Compiled by Indrajeet Yadav'.
- STATEMENTS: S1-S20, each with a space marked for True/False and a one-line justification.
- ANSWER KEY: for each statement, the correct verdict plus a one-line reason or counter-example, clearly separated at the end.

RULES: Each statement must be unambiguously true or false. Justifications in the key must be correct. Balanced mix. Simple wording.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block.
