Last-Minute Revision Capsule Chapter

A compact revision chapter covering a whole unit for fast exam prep.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. List the chapters to compress.
  2. 2. State class and board.
  3. 3. Send the prompt.
  4. 4. Check the formula list for accuracy.
🔧 If it is not right, reply with this: If it is too long, reply: "Cut this to two pages, keep only exam-critical formulas and traps, and remove the worked derivations."
THE PROMPT
ROLE: Act as an exam coach writing a high-yield revision capsule.

CONTEXT: I need a last-minute revision capsule for [CLASS/GRADE] students of [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET] covering [UNIT OR LIST OF CHAPTERS]. Students will read it the day before the exam, so it must be dense, accurate, and high-yield.

YOUR TASK:
1. Compress the whole unit into a short revision capsule.
2. For each chapter, give the key formulas, key definitions, and key methods only.
3. Add a short list of the most common exam traps and how to avoid them.
4. Add 5-8 "one-mark facts" that frequently appear in exams.
5. Add a 10-point final checklist a student should be able to tick before the exam.

INPUT: Here are the chapters and any exam pattern details: [PASTE HERE].

HOW TO WORK IT OUT: Identify the highest-yield content internally first. VERIFY each formula two ways: (a) confirm it is standard and correct, and (b) state its condition of use so it is not misapplied. Confirm each exam trap is a real, common one. Never invent formulas or values. Keep all mathematics in clean LaTeX or clear plain text and keep the capsule tight.

OUTPUT FORMAT (exact order):
1. Heading: "Revision Capsule"
2. Per Chapter: Formulas (with conditions), Definitions, Methods
3. Common Exam Traps and Fixes
4. One-Mark Facts
5. Final 10-Point Checklist

RULES: Dense and high-yield, no filler. Every formula shows its condition. Real traps only. Must be short enough to read in one sitting.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
