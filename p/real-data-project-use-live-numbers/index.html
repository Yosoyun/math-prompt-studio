Real-Data Project (Use Live Numbers)

A project using real public data, with where-to-find-it and what-to-calculate.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Pick a real-world theme and class.
  2. 2. Paste the prompt into any AI chat.
  3. 3. Read where students find the data.
  4. 4. Print the calculation steps.
  5. 5. Grade with the rubric provided.
🔧 If it is not right, reply with this: If a data source is unreliable, reply: "Suggest only well-known, free public sources, and add a fallback dataset students can use if a website is blocked."
THE PROMPT
ROLE: Act as a maths teacher who connects classroom maths to real, publicly available data.

CONTEXT: I teach [CLASS/GRADE] under [BOARD e.g. CBSE / ICSE / State Board]. I want a REAL-DATA PROJECT on the theme [THEME e.g. weather/rainfall / cricket scores / population / prices / sports statistics]. Students should find real numbers, organise them, and apply [MATHS SKILL e.g. averages / percentages / ratios / graphs] my class already knows.

YOUR TASK:
1. Pose one real-world question students will answer with data.
2. Suggest 2-3 reliable, free places to find the data (described clearly, no paid sites).
3. Tell students exactly which numbers to record and in what table.
4. List the calculations to perform, matched to the class level.
5. Provide a fully worked SAMPLE with realistic illustrative numbers and every calculation shown.
6. Include a fallback dataset (a small ready table) in case students cannot access a source.
7. Provide a 20-mark rubric.

INPUT: Here is my theme and the maths skill to apply: [PASTE THEME AND SKILL HERE]. If blank, choose a strong, accessible theme for the class and board above.

HOW TO WORK IT OUT: Plan internally. Use realistic but clearly labelled illustrative numbers for the sample (do not present them as today's real figures). VERIFY every calculation a second way and show the check. Never invent a data source you are unsure exists; describe the type of source plainly. Keep all maths legible in clean plain text or simple LaTeX.

OUTPUT FORMAT (use these exact headings in this order):
1. PROJECT TITLE
2. THE REAL-WORLD QUESTION
3. WHERE TO FIND THE DATA (2-3 sources described)
4. WHAT TO RECORD (table headings)
5. CALCULATIONS TO DO (step list)
6. WORKED SAMPLE (illustrative data + calculations + checks)
7. FALLBACK DATASET (a ready small table)
8. RUBRIC (20 marks)

RULES: Free sources only. Be honest about illustrative vs real numbers. Short, plain sentences.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title block at the top.
