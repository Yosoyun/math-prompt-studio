Turn a Photo of a Real Object Into Maths Problems

Real-life problems built from your own photo of an object, bill or scene.

Attach a photo of the question first  ·  Best tool:  ChatGPT or Gemini (can read photos)

✅ How to use this effectively

  1. 1. Take a clear, well-lit photo.
  2. 2. Attach it in the chat.
  3. 3. Paste the prompt and your class.
  4. 4. Send and read the problems.
  5. 5. Ask for more if the photo has more data.
🔧 If it is not right, reply with this: If the AI misreads the photo, reply: 'Re-read the photo carefully, list every number you can see, then redo and verify each answer.'
THE PROMPT
ROLE: Act as a maths teacher who creates real-life problems from a real photograph.
CONTEXT: My students are in [CLASS/GRADE] under [BOARD or EXAM]. I have a photo of something real (a shop bill, a water tank, a tiled floor, a signboard with distances, a packet with weight and price, a field). I want problems built directly from what is in the photo so the maths is undeniably real. Topic focus if any: [TOPIC e.g. percentages / mensuration / ratios].
YOUR TASK:
1. First, list every number, measurement, price, or shape you can clearly read in the photo.
2. If anything is unreadable, say so and ask me to type it, OR make a clearly labelled reasonable assumption.
3. Create 6 problems using ONLY the data from the photo (plus any labelled assumption): totals, percentages, area or perimeter, unit price, scaling.
4. Give a full solution for each.
INPUT: I have attached a photo of the object/bill/scene. (If the photo is unclear, tell me exactly what to retype.)
HOW TO WORK IT OUT: Read the photo carefully and extract the data first. Solve internally, then VERIFY each answer by two independent checks (recompute, and sanity-check the result). Use ONLY what is visible in the photo plus clearly labelled assumptions; never invent figures. Keep units and all maths legible.
OUTPUT FORMAT:
- 'What I See in the Photo' (a list of every readable number and shape).
- 'Problems from the Photo' numbered 1 to 6 with marks.
- 'Solutions' (full working with units, each ending with 'Check:').
- 'Note' (anything unreadable I should retype).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
RULES: Only use visible data, label every assumption, correct units, simple language, class level only.
SIGNATURE: At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block at the top.
