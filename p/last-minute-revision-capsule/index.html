Last-Minute Revision Capsule

A fast revision capsule of only the high-yield points before a test.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. List the chapter or chapters.
  2. 2. Paste the prompt.
  3. 3. Skim the capsule for gaps.
  4. 4. Ask to add anything missing.
🔧 If it is not right, reply with this: If something important is missing, reply: "Add [missing point] to the right section and verify any formula you state."
THE PROMPT
ROLE: Act as a senior mathematics teacher who creates last-minute revision capsules that maximise marks.

CONTEXT: My [CLASS/GRADE] students have a test on [CHAPTER NAME(S)] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. They have very little time. I need a high-yield revision capsule: only the points most likely to fetch marks, nothing decorative.

YOUR TASK:
1. Pull out the highest-yield formulas, results, and standard tricks for these chapters.
2. Flag the 5 most commonly tested question types and the quickest correct approach for each.
3. List the typical mistakes that cost marks.
4. Add one 30-second memory hook per formula group where it genuinely helps.

INPUT: Here are the chapters: [CHAPTER NAME(S)]. If I paste the exact syllabus or past-paper pattern, prioritise according to it.

HOW TO WORK IT OUT: Decide importance based on standard exam weighting for this board or exam. Solve any quick example internally and VERIFY each result two ways before listing it. Never invent a formula, a trick, or a value. Keep all mathematics legible in clean LaTeX or clear plain text. If you are unsure a trick is standard, leave it out.

OUTPUT FORMAT (in this order):
1. Capsule Title: [CHAPTER NAME(S)] - Revision Capsule
2. Must-Remember Formulas (grouped, each one line)
3. Top 5 Question Types and Fast Approaches
4. Mark-Saving Mistakes to Avoid (6 bullets)
5. Memory Hooks (only where useful)
6. Final 60-Second Checklist (5 ticks)

RULES: Be ruthless about high-yield only. No long explanations. Everything must be standard and exam-correct. Keep it short enough to read in 10 minutes. State one assumption line if the chapter scope is unclear.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.

SIGNATURE: End your ENTIRE response with exactly this footer block (keep the lines and symbols):
