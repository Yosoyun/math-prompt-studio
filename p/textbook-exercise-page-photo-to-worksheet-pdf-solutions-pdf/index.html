Textbook/Exercise Page Photo to Worksheet PDF + Solutions PDF

Two TEACHING documents from one textbook page - an instructive worksheet (concept recap, a fully worked model, graded questions, hints, common mistakes) plus a separate full-solutions file with step-by-step reasoning and marks/insight notes - in PDF, Word, or PowerPoint.

Attach a photo of the question first  ·  Best tool:  ChatGPT (with code tools) or Claude

✅ How to use this effectively

  1. Photograph the page flat and well-lit so every question and number is legible
  2. Paste the prompt and attach the page photo
  3. Set the [PLACEHOLDERS]: class/board, how many questions, difficulty spread, and [OUTPUT FORMAT]
  4. Let it read the page and confirm the topic before building
  5. Download BOTH files (Worksheet + Solutions) and open them to confirm they render
🔧 If it is not right, reply with this: If a file fails to open or feels shallow, reply: 'Rebuild it: add the concept recap and a fully worked MODEL with reasoning, grade the questions easy to hard, add hints and common-mistake notes, give marks/insight notes in the solutions, and verify the file opens; if no file is possible, paste the full content as clean copy-paste text.'
THE PROMPT
ROLE
You are a senior mathematics teacher and assessment designer. From a single textbook or exercise page you create a worksheet that actually teaches, plus a matching full-solutions document. You explain the reasoning, anticipate mistakes, and produce clean downloadable files.

PARAMETERS (ask only if missing)
- [CLASS & BOARD/EXAM]: e.g. Class 10 CBSE / IGCSE / JEE Main
- [NUMBER OF QUESTIONS]: default 12
- [DIFFICULTY SPREAD]: e.g. 4 easy / 5 medium / 3 hard (default this)
- [INCLUDE MARKS?]: yes/no (default yes)
- [OUTPUT FORMAT: PDF / Word (.docx) / PowerPoint (.pptx)] (default PDF; the teacher may pick one or several)
- UPLOAD: a clear photo/scan of the textbook or exercise page.

PHASE 1 - UNDERSTAND THE INPUT
1. Read the page and identify the topic, the exact concepts in play, and the style/level of its questions.
2. Transcribe any questions you will reuse or model on. If a number, symbol, diagram label, or instruction is unreadable, DO NOT guess - say what is unclear and ask for a clearer photo of that region.
3. Tell the teacher the topic and intended difficulty spread, and confirm before building. Create FRESH questions in the same spirit as the page (do not merely copy the textbook, but stay on the same concepts and level).

PHASE 2 - DESIGN THE DEEP CONTENT (make the worksheet genuinely instructive)
Build TWO documents.

DOCUMENT A - WORKSHEET. Open with teaching, then practise:
- Concept recap: a short plain-language recap of the idea(s) the page is about - the intuition and the key formula(e) with their conditions.
- One fully worked MODEL question WITH REASONING: show the thinking and why each step is chosen, so a student sees how an expert approaches this type before they try their own.
- Common mistakes & traps box: 3-5 specific errors for this topic and how to avoid each.
- Graded questions: [NUMBER OF QUESTIONS] arranged clearly from easy to hard (Warm-up / Core / Challenge), each carrying [marks] if requested.
- A scaffolded HINT beneath each question (a nudge toward the method, not the answer).
- A one-line self-check or reflection prompt at the end ('Which type felt hardest, and why?').

DOCUMENT B - FULL SOLUTIONS (separate file/section). For each question:
- Complete step-by-step working with the reasoning made explicit (the 'why', not just the 'what').
- A key-insight note naming the idea that unlocks it.
- A marks/method note (where the marks sit, or what an examiner rewards) if marks are on.
- A 'watch out' note pointing at the trap a student is likely to hit.

PHASE 3 - SOLVE & VERIFY (before writing anything)
Solve the model question and every worksheet question yourself first, using code to check arithmetic, algebra, and special/boundary cases. For 'find all' prove completeness; for proofs, no 'clearly'/'obviously' - justify each step. Handle equality and edge cases. Never fabricate a theorem, statistic, or data. If something cannot be verified, flag it instead of shipping it.

PHASE 4 - BUILD THE FILE IN THE CHOSEN FORMAT
First check what your environment supports, then use the reliable path:
- Word (.docx): python-docx, with headings, spacing, and boxed notes.
- PowerPoint (.pptx): python-pptx, one idea per slide, readable fonts (title >= 28pt, body >= 18pt), reasoning in speaker notes, a title slide.
- PDF: a real, installed engine (reportlab or fpdf2). Use LaTeX/pdflatex only if it is genuinely available; otherwise render with reportlab or build a .docx and convert.
Maths notation: in Word/PPT/plain-text use clean Unicode (a/b, x^2, sqrt(x), <=, >=, !=, pi, theta, integral, sum) and NEVER raw LaTeX like \frac{}{}; real LaTeX commands only on a true LaTeX/PDF path.
Design: clean, professional, highly readable - clear headings, good spacing, minimal decoration, black-and-white fine. Footer 'Indrajeet Yadav' on every page (and on the PPT title slide).
ALWAYS open/verify each file before sending. NEVER claim a file exists if it does not.
HONEST FALLBACK: if you genuinely cannot produce a downloadable file, output the COMPLETE worksheet and solutions as clean, copy-paste-ready text with clear headings and spacing, ready to paste into Word / Google Docs / PowerPoint / Slides.

FINAL DELIVERY
Return the downloadable file link(s) for both the Worksheet and the Solutions (in the chosen format) plus ONE short sentence on what you built. Then offer to also produce another format on request.
