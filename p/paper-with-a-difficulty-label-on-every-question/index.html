Paper with a Difficulty Label on Every Question

A full paper where each question is tagged Easy, Medium or Hard, plus a difficulty summary table.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Paste the prompt into a chat.
  2. 2. Set your Easy/Medium/Hard split.
  3. 3. List the chapters to cover.
  4. 4. Send and check each question's label.
  5. 5. Read the summary table to confirm the mix.
  6. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If the mix is off, reply: 'The actual difficulty split does not match my target. Adjust the questions so the summary table matches [your split], then re-verify the key.'
THE PROMPT
ROLE: Act as a mathematics examiner who tags questions by difficulty for balanced papers.

CONTEXT: I need a full paper for Class [CLASS/GRADE], [BOARD or EXAM e.g. CBSE / ICSE / State Board], where EVERY question carries a difficulty label: Easy, Medium or Hard. My target difficulty split is [e.g. 40 percent Easy, 40 percent Medium, 20 percent Hard] by marks. Total marks [TOTAL MARKS], time [TIME].

YOUR TASK:
1. Write a complete paper covering the chapters below.
2. Beside every question, show its marks and a difficulty tag in square brackets, e.g. '[Medium]'.
3. Hit the target difficulty split as closely as possible by marks.
4. Provide an answer key and a DIFFICULTY SUMMARY TABLE showing marks and percentage for Easy, Medium and Hard.

INPUT: Chapters or topics to cover: [LIST CHAPTERS or TOPICS]. Use only these.

HOW TO WORK IT OUT: Solve every question internally first, then verify each final answer twice by independent methods. Honestly rate each question's difficulty, then add up the marks per difficulty and confirm they match the target split. Never invent formulas or values. Keep mathematics legible in clean LaTeX or plain text. Confirm marks total exactly [TOTAL MARKS].

OUTPUT FORMAT, in this order:
1. HEADER BLOCK: paper title, class, board, time, maximum marks, 'Compiled by Indrajeet Yadav'.
2. INSTRUCTIONS: numbered, explaining the difficulty tags.
3. THE PAPER: numbered questions, each ending with marks and a '[Easy/Medium/Hard]' tag.
4. DIFFICULTY SUMMARY TABLE: marks and percentage in each band.
5. ANSWER KEY: final answers plus short outlines for long questions.

RULES: Every question must be tagged. The summary must match the target split closely. Stay inside the listed syllabus. Number every question. Marks total exactly.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
