Text Mind Map of Any Chapter

A clean indented text mind map of one chapter, ready to copy into notes.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Paste the prompt below.
  3. 3. Fill in chapter, class and board.
  4. 4. Send and read the indented map.
  5. 5. Ask for more detail on weak branches.
🔧 If it is not right, reply with this: If a branch is too thin, reply: "Expand the [branch name] branch with 3 to 5 sub-points and one example each." If it added wrong formulas, reply: "Remove any formula you are not 100 percent sure of and re-verify the rest."
THE PROMPT
ROLE: Act as a master mathematics teacher and a curriculum mind-map expert.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. I need a clean, well-organised TEXT mind map of one chapter that I can paste into my notes and read aloud in class. No image is needed - this is plain text only.

YOUR TASK:
1. Build a hierarchical text mind map of the chapter I name.
2. Use the central topic as the root, then 4 to 7 main branches, then sub-branches under each.
3. Cover: key definitions, main concepts, important formulas, standard methods, common mistakes, and typical exam question types.
4. Keep it tight - this is a revision map, not a textbook.

INPUT: Here is the chapter and any sub-topics I already have: [PASTE CHAPTER NAME AND ANY SUB-TOPICS HERE]. If I paste nothing extra, use the standard [BOARD] syllabus for that chapter at [CLASS/GRADE] level.

HOW TO WORK IT OUT: First plan the full chapter coverage privately. Then organise it from general to specific. VERIFY every formula and definition twice before including it - once against standard syllabus logic and once by a quick sanity check (units, special cases, a simple test value). Never invent a formula, rule, or value; if unsure, write "(please verify)" next to it instead of guessing. Keep all mathematics fully legible in clear plain text (for example write x^2, sqrt(x), (a+b)/c) so it reads correctly even without special symbols.

OUTPUT FORMAT (use exactly this, in this order):
CENTRAL TOPIC: <chapter name>
Branch 1 - Definitions and Key Terms
   - sub-point
   - sub-point
Branch 2 - Core Concepts
   - sub-point
Branch 3 - Formulas (write each formula clearly)
   - sub-point
Branch 4 - Methods and Steps
   - sub-point
Branch 5 - Common Mistakes
   - sub-point
Branch 6 - Exam Question Types
   - sub-point
Use indentation with dashes so the hierarchy is obvious. Keep each sub-point to one line.

RULES: Be accurate above all. No filler, no repetition, no decorative symbols beyond simple dashes. Match the depth to [CLASS/GRADE]. Total length should fit comfortably on one to two pages.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block at the top.
