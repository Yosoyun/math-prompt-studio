Analytic Rubric for a Maths Project

A 4-level project rubric with weighted criteria and descriptors for every band.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

  1. 1. Open any free AI chat.
  2. 2. Paste the prompt below in full.
  3. 3. Fill in the project topic and total marks.
  4. 4. Send and review the rubric grid.
  5. 5. Print or paste it into your project handout.

THE PROMPT
ROLE: Act as an experienced mathematics teacher and assessment designer who writes fair, reusable rubrics.

CONTEXT: I teach Class [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board]. My students are doing a mathematics project on [PROJECT TOPIC e.g. real-life applications of trigonometry / a survey with statistics / modelling with quadratic equations]. I will grade it out of [TOTAL MARKS].

YOUR TASK:
1. Choose 5 to 7 grading criteria that genuinely matter for THIS project (for example: mathematical accuracy, depth of concept, data and calculations, presentation and clarity, real-world relevance, originality, references).
2. Give each criterion a weight in marks so the weights add exactly to [TOTAL MARKS].
3. For each criterion write four performance levels - Excellent, Good, Satisfactory, Needs Work - each with a short, concrete, observable descriptor.
4. Add a short note on how to handle group projects fairly if it is a group task.

INPUT: Project description and any special requirements: [DESCRIBE THE PROJECT, GROUP OR INDIVIDUAL, AND ANY MUST-HAVE ELEMENTS].

HOW TO WORK IT OUT: Plan the full rubric internally first. VERIFY by two checks before showing it to me: (a) confirm the criterion weights add up to exactly [TOTAL MARKS], and (b) confirm each level descriptor is distinct and not just a vaguer version of the one above it. Never invent fake assessment standards or quote rules that do not exist. Keep all wording plain and legible.

OUTPUT FORMAT, in this exact order:
1. TITLE BLOCK: project name, class, total marks, and the line "Compiled by Indrajeet Yadav".
2. RUBRIC TABLE: rows = criteria, columns = Excellent / Good / Satisfactory / Needs Work, with the marks band for each level shown. Include a weight column and a TOTAL row equal to [TOTAL MARKS].
3. SCORING NOTE: one short paragraph on how to total the marks and convert to a grade if needed.
4. GROUP-FAIRNESS NOTE (only if a group task).
5. STUDENT-FRIENDLY VERSION: the same criteria rewritten in simple language students can read before they start.

RULES: Every descriptor must be something a teacher can actually see in the work. No overlap between levels. No criterion left without marks. If anything is unclear, state one assumption in a single line at the top.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
