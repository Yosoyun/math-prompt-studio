Full Mock Test with Complete Solutions

A complete mock test plus a separate, fully worked solution set.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Type your class, board, and total marks.
  2. 2. List the chapters to be covered.
  3. 3. Paste the prompt and send.
  4. 4. Ask for solutions if they stop early.
  5. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If a question looks wrong or unsolvable, reply: "Re-check question 7 - confirm it has a clean answer, then verify that answer two different ways."
THE PROMPT
ROLE: Act as a senior mathematics paper-setter and examiner.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. I need a complete, original mock test that students can sit in an exam hall, plus a separate solution set I can keep.

YOUR TASK:
1. Build ONE full mock test for these chapters: [LIST CHAPTERS]. Total marks: [TOTAL MARKS]. Suggested time: [TIME e.g. 3 hours].
2. Use a sensible question split: some 1-mark, some 2-mark, some 3-mark, and some 5-mark questions, so the marks add up exactly to the total.
3. Make every question ORIGINAL (do not copy famous textbook questions) but keep the style and difficulty exactly matching [BOARD or EXAM].
4. After the paper, produce a SEPARATE full solution set with complete steps for every question.

INPUT: Here are the chapters and marks: [PASTE CHAPTER LIST AND TOTAL MARKS HERE]. If you need anything else, ask me ONE short question first, then proceed.

HOW TO WORK IT OUT: Solve every question yourself internally BEFORE printing it. For each final answer, VERIFY it by two independent checks (for example: substitute the answer back, or solve by a second method, or estimate and compare). Never invent formulas or values. If any question does not give a clean, checkable answer, silently replace it with a better one. Keep all mathematics fully legible - use clear plain text or clean LaTeX, and never leave a step as 'obvious'.

OUTPUT FORMAT (use these exact headings, in this order):
SECTION 0 - PAPER HEADER: Subject, Class, Board/Exam, Total Marks, Time, and General Instructions (5 short rules).
SECTION A - 1 Mark Questions (numbered)
SECTION B - 2 Mark Questions (numbered)
SECTION C - 3 Mark Questions (numbered)
SECTION D - 5 Mark Questions (numbered)
MARKS MAP: a small table showing how the marks total to [TOTAL MARKS].
--- PAGE BREAK ---
SOLUTIONS: full worked solution for every question, in the same numbering, with the final answer boxed or clearly labelled 'Answer:'.
VERIFICATION NOTE: one line per section confirming you checked the answers.

RULES: No repeated questions. No answer left unverified. Difficulty must feel like a real [BOARD or EXAM] paper, not harder or easier. If marks do not add up, fix them before showing me.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
