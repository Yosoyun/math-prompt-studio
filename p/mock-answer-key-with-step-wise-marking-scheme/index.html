Mock Answer Key with Step-Wise Marking Scheme

A marking scheme that shows where each mark is awarded, examiner-style.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Paste your paper, or ask it to make one.
  2. 2. State board and total marks.
  3. 3. Send the prompt.
  4. 4. Use the scheme to grade fairly.
🔧 If it is not right, reply with this: If marks per step don't add up, reply: "For question 5, the step marks must total the question marks - recount and fix."
THE PROMPT
ROLE: Act as a board examiner who writes official-style step-wise marking schemes.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board]. When I grade, I need to know exactly where each mark is given, like a real board marking scheme, so grading is fair and consistent.

YOUR TASK:
1. Use the paper I provide (or, if I provide none, build a full mock for: [LIST CHAPTERS], total [TOTAL MARKS]).
2. For EVERY question, write a STEP-WISE marking scheme: list each step a student must show and the marks for that step, so the step marks add up to the question's total marks.
3. Note where 'method marks' are given even if the final answer is wrong, and where to give 'follow-through' credit.
4. Give the final answer for each question too.

INPUT: Here is my paper: [PASTE THE PAPER HERE]. (If empty, build one from the chapters above.)

HOW TO WORK IT OUT: Solve every question internally first. VERIFY each final answer two independent ways. Never invent formulas or values. For each question, confirm the step marks total the question marks. Keep all maths legible in clean plain text or LaTeX.

OUTPUT FORMAT (exact order):
MARKING SCHEME: for each question, a block:
  Q. No. and marks.
  Step 1 ... (marks) / Step 2 ... (marks) / ... so steps sum to the question total.
  Final Answer: labelled.
  Examiner note: where to give method marks or follow-through credit.
TOTALS CHECK: a line confirming all question marks sum to [TOTAL MARKS].
VERIFICATION NOTE: confirm answers and mark sums checked.

RULES: Step marks must total each question's marks; question marks must total [TOTAL MARKS]. Be specific about what earns each mark. Style true to [BOARD or EXAM].

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
