Concept Hierarchy Tree

A top-down tree showing how big ideas split into smaller sub-concepts.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Type the topic and class level.
  3. 3. Paste the prompt below.
  4. 4. Send and read top to bottom.
  5. 5. Ask to split any node further.
🔧 If it is not right, reply with this: If the tree is too shallow, reply: "Go one more level deeper under every leaf and add a one-line example at the bottom level." If a branch is mislabeled, reply: "Fix that node and re-check the level above it."
THE PROMPT
ROLE: Act as a mathematics teacher and a concept-mapping specialist who builds clean hierarchy trees.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE]. I want a CONCEPT HIERARCHY TREE (text form) for one topic, showing how the big idea breaks down into categories, then sub-types, then specific cases. This helps students see structure. No image needed.

YOUR TASK:
1. Put the main concept at the top (Level 1).
2. Break it into 3 to 5 major categories (Level 2).
3. Break each category into sub-types (Level 3).
4. At the lowest level, give the specific cases, rules, or formulas with one short example each.
5. Keep the tree balanced and logical.

INPUT: Here is the topic I want as a tree: [PASTE TOPIC HERE]. If I give only the topic name, use standard [BOARD] content for [CLASS/GRADE].

HOW TO WORK IT OUT: First plan the whole classification privately so the categories do not overlap and nothing important is missing. VERIFY each rule, formula and example twice (syllabus accuracy plus a quick sanity check with a test value). Never invent rules; mark anything uncertain "(verify)". Write all maths in clean plain text (a^2, |x|, log x).

OUTPUT FORMAT (use indentation to show levels):
LEVEL 1: <main concept>
  LEVEL 2: Category 1
     LEVEL 3: sub-type
        - specific case / formula - short example
     LEVEL 3: sub-type
        - specific case / formula - short example
  LEVEL 2: Category 2
     ...
Every lowest node must end with one tiny worked example.

RULES: Categories must be mutually exclusive and collectively cover the topic. No overlap, no gaps, no padding. Match depth to [CLASS/GRADE].

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
