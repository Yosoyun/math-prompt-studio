Difficulty-Tiered Quiz (Bronze / Silver / Gold)

One topic, three difficulty tiers of questions so every student is challenged at the right level.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Fill class, board and chapter.
  2. 2. Say how many questions per tier.
  3. 3. Send the prompt.
  4. 4. Assign tiers by student readiness.
  5. 5. Grade using the tier-wise key.
🔧 If it is not right, reply with this: If Gold feels the same as Silver, reply: 'Make the Gold tier clearly harder (multi-step or twist) than Silver, then re-verify all answers.'
THE PROMPT
ROLE: Act as a mathematics teacher who differentiates practice by ability level.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. The chapter is [CHAPTER NAME]. I want a tiered quiz with [NUMBER, e.g. 5] questions in each of three tiers: BRONZE (easy / building confidence), SILVER (standard exam level), GOLD (challenge / higher-order).

YOUR TASK:
1. Create [NUMBER] questions per tier on the same sub-topic, so all three tiers connect but clearly differ in difficulty.
2. BRONZE: single-step, direct application. SILVER: typical exam, two steps. GOLD: multi-step or a twist that needs deeper insight.
3. Use a mix of MCQ and short-answer as fits each tier; mark which is which.

INPUT: Here is the sub-topic to focus on: [PASTE TOPIC OR LEAVE BLANK].

HOW TO WORK IT OUT: Solve every question and VERIFY each answer by a second method. Make sure each tier is genuinely harder than the last (state one reason why each Gold question is harder). Do not invent formulas. Keep maths legible.

OUTPUT FORMAT:
Three clearly headed sections: BRONZE, SILVER, GOLD. Under each, the numbered questions.
ANSWER KEY: organised by tier, each with a one-line worked answer.
TEACHER NOTE: one line on which tier to give which type of student.

RULES: Same sub-topic across tiers. Difficulty must clearly increase. No tier should be mislabelled. Suited to the class above.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block at the top.
