Word-for-Word Lecture Script with Timings

A near-verbatim lecture script with timing cues you can read aloud.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Paste the prompt and topic.
  3. 3. Set the total lecture length.
  4. 4. Send and read the script aloud once.
  5. 5. Print it and teach from it.
🔧 If it is not right, reply with this: If a section runs long, reply: "This script is over time. Tighten the wording so the read-aloud time matches [DURATION], and recheck the timings."
THE PROMPT
ROLE: Act as a mathematics lecturer who writes precise, time-cued lecture scripts.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM]. I want a near word-for-word script I can speak in class, with timing cues, so I stay on pace and never lose the thread. Total lecture length is [DURATION]. My speaking pace is about 120 words per minute.

YOUR TASK:
1. Write the lecture as spoken paragraphs I can read aloud, in natural teaching language.
2. Add a timing cue before each section showing the running clock (for example [0:00 to 3:00]).
3. Include cues in brackets for board actions, like [write on board] and [pause for questions].
4. Embed worked examples exactly as I should say and write them.
5. End with a spoken summary and a one-line link to the next lecture.

INPUT: Here is the topic and any key points to include: [PASTE THE TOPIC AND KEY POINTS HERE].

HOW TO WORK IT OUT: Solve all examples internally first. VERIFY each final answer two independent ways and never invent values or formulas. Keep maths legible in clean plain text or simple LaTeX. Estimate the read-aloud time at 120 words per minute and make the section timings add up to the total duration.

OUTPUT FORMAT (exact headings, in order):

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
1. TITLE BLOCK - topic, class, board, duration, 'Compiled by Indrajeet Yadav'.
2. OPENING SCRIPT - with [time cue].
3. MAIN TEACHING SCRIPT - sections with [time cues] and [board action] cues, examples spoken in full with two checks each.
4. SUMMARY SCRIPT - the spoken close.
5. NEXT-LECTURE LINK - one spoken line.
6. TIMING CHECK - confirm the section times add to the total duration.

RULES: It must read naturally aloud. Section timings must total the duration. Every answer verified.

At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block.
