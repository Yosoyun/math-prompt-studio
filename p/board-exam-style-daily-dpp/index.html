Board-Exam-Style Daily DPP

A daily sheet that mirrors your board's exact question types, weightage and language.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Name the exact board and class.
  2. 2. Set the chapter.
  3. 3. Paste the prompt and send.
  4. 4. Print Section A for students.
  5. 5. Grade with the step-marking scheme.
🔧 If it is not right, reply with this: If the style feels off, reply: "Make the wording and question types match the latest [BOARD] paper pattern more closely."
THE PROMPT
ROLE: Act as a board-exam paper-setter who knows the official pattern, wording and step-marking of Indian school boards.

CONTEXT: I want a daily BOARD-STYLE DPP that mirrors the real exam for [CLASS/GRADE] students of [BOARD e.g. CBSE / ICSE / specific State Board]. Chapter = [CHAPTER]. 

YOUR TASK:
1. Create a sheet using the board's actual question forms for this chapter: very-short-answer, short-answer, long-answer, and a case-study/competency question if the board uses them.
2. Match the board's typical mark weightage and the exact tone of board questions.
3. Provide a step-marking scheme exactly as examiners award marks.
4. Add hints, an answer key, and model answers written the way a topper would present them.

INPUT: Chapter = [CHAPTER]. Class = [CLASS/GRADE]. Board = [BOARD]. If unclear, assume and state it.

HOW TO WORK IT OUT: Solve every question yourself first. Verify each final answer by two independent checks. Match the board's real difficulty and phrasing. Never invent formulas or values. Keep all maths legible in clean LaTeX or clear plain text.

OUTPUT FORMAT, in this exact order:
1. TITLE BLOCK: 'Board-Style DPP - [CHAPTER] - [BOARD]' with class, total marks, and 'Compiled by Indrajeet Yadav'.
2. SECTION A - QUESTIONS: grouped by type (Very Short, Short, Long, Case-Study) with marks shown, printable.
3. SECTION B - HINTS: one one-line hint per question.
4. SECTION C - STEP-MARKING SCHEME: how each mark is awarded, board-style.
5. SECTION D - MODEL ANSWERS: clean topper-style full solutions.

RULES: Question types and wording must match the named board. Step-marking must reflect official examiner practice. Marks must total correctly.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
