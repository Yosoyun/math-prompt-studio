Prerequisite Map: What to Teach First

An ordered list of what a student must already know before this concept.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Paste the prompt.
  3. 3. Add the concept and class level.
  4. 4. Send and read the ordered prerequisites.
  5. 5. Use the quick check to find gaps.
🔧 If it is not right, reply with this: If the order seems wrong, reply: "Re-order the prerequisites strictly so each item only needs items above it, and explain the dependency for each."
THE PROMPT
ROLE: Act as a curriculum-design expert and mathematics teacher.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. Before I teach this target concept, I need to know exactly what students must already understand: [WRITE THE TARGET CONCEPT HERE].

YOUR TASK:
1. List every prerequisite skill or concept needed, in strict learning order.
2. For each prerequisite, write one line on WHY it is needed for the target.
3. Mark each prerequisite as ESSENTIAL or HELPFUL.
4. Give one quick check question per ESSENTIAL prerequisite to test readiness.
5. Suggest the single most common missing prerequisite that causes failure.

INPUT: Here is the target concept: [PASTE OR TYPE IT HERE]. Class level: [CLASS/GRADE].

HOW TO WORK IT OUT: Order the list so that each item depends only on items above it; internally check the chain for circular dependencies. VERIFY readiness questions by solving them fully and confirming answers two ways. Never invent prerequisites that are not genuinely required, and never invent formulas. Keep maths legible in clear plain text.

OUTPUT FORMAT (use these exact headings):
- TARGET CONCEPT
- ORDERED PREREQUISITES (a numbered list; each item shows: name | ESSENTIAL or HELPFUL | why it is needed in one line)
- READINESS CHECK (one short question per ESSENTIAL item, with answers)
- THE USUAL MISSING LINK (the one gap that most often blocks students, and how to spot it)

RULES: Strict dependency order; no item should require something listed later. Be concrete to this exact concept. Keep readiness questions short and answerable in under a minute.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
