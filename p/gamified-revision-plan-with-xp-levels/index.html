Gamified Revision Plan With XP & Levels

A week-by-week quest plan with XP, badges, a boss-battle mock test, a comeback rule and a student tracker.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Type your class, board and weeks to exam.
  2. List the chapters to revise.
  3. Paste the prompt and send.
  4. Share the XP and levels table.
  5. Hand out the one-page tracker.
🔧 If it is not right, reply with this: If a week looks impossibly heavy or XP doesn't add up, reply: "Re-balance the weekly tasks so the syllabus fits the time, and recompute the XP totals so the level thresholds are correct."
THE PROMPT
ROLE: Act as a mathematics teacher and study-coach who turns boring exam revision into a gamified, week-by-week challenge.

CONTEXT: I teach [CLASS/GRADE] preparing for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET], with the exam in about [NUMBER] weeks. I want a gamified revision plan covering [LIST CHAPTERS or 'the whole syllabus'] that students actually want to follow.

YOUR TASK:
1. Break the revision into weekly 'quests', each with a chapter focus, a clear goal, and XP (experience points) for completing tasks.
2. Define levels/badges students unlock as XP grows (for example Bronze, Silver, Gold solver).
3. For each week give 3 concrete tasks (practice set, self-test, error log) and the XP each is worth.
4. Add a boss-battle mock test at the end with a target score, and a comeback rule for students who fall behind.
5. Provide a one-page tracker layout students fill in themselves.

INPUT: Class, board, weeks to exam and chapters are above. If I add nothing, spread the syllabus evenly across the weeks.

HOW TO WORK IT OUT: Check the plan's arithmetic: confirm total XP, the level thresholds, and that finishing the weekly tasks realistically covers the chapters in the time given. State the XP totals you used. Never promise unrealistic coverage. Keep all numbers in clear plain text.

OUTPUT FORMAT (use these exact headings):
PLAN OVERVIEW (2-3 lines)
XP AND LEVELS TABLE (badge, XP needed)
WEEK-BY-WEEK QUESTS (each week: chapter focus, 3 tasks with XP, goal)
BOSS-BATTLE MOCK TEST (target score, when)
COMEBACK RULE (for students who fall behind)
ONE-PAGE STUDENT TRACKER (describe the columns to draw)
KICK-OFF PEP TALK (short paragraph to read aloud)

RULES: XP totals and level thresholds must add up correctly. The plan must realistically fit the weeks available. No week may be impossibly heavy. Keep the tracker simple enough to draw by hand.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
