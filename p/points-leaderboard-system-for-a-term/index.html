Points & Leaderboard System for a Term

A fair points scheme with weekly caps, a Most Improved track, low-cost rewards and a worked example.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Type your class, board and term length.
  2. Paste the prompt and send.
  3. Draw the leaderboard with a Most Improved column.
  4. Announce point values to the class.
  5. Update the board once a week.
🔧 If it is not right, reply with this: If one activity lets a student run away with it, reply: "Lower the points or add a weekly cap on that activity, then re-run the 3-student worked example to prove it stays fair."
THE PROMPT
ROLE: Act as a mathematics teacher and classroom-motivation designer who builds a fair, term-long points and leaderboard system.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET] over a term of about [NUMBER] weeks. I want a points and leaderboard system that rewards effort and improvement in maths, not just raw talent, so weaker students stay motivated.

YOUR TASK:
1. Design a points system covering homework done, class participation, quiz scores, helping a peer, and personal-best improvement.
2. Give exact point values and weekly caps so no single activity dominates.
3. Design a simple leaderboard layout I can keep on paper or a whiteboard, including a 'most improved' track so newcomers can win.
4. Suggest weekly and end-of-term rewards that cost little or nothing.
5. Add 3 anti-cheating and fairness rules.

INPUT: Class, board and term length are above. If I add nothing, assume a class of about 40 with limited budget.

HOW TO WORK IT OUT: Internally simulate a few weeks for a strong, an average and a weak student to confirm the maths of the points keeps the system motivating and that 'most improved' is genuinely winnable. State the totals from that check. Never invent unrealistic numbers. Keep all arithmetic transparent and in clear plain text.

OUTPUT FORMAT (use these exact headings):
SYSTEM OVERVIEW (2-3 lines)
POINTS TABLE (activity, points, weekly cap)
LEADERBOARD LAYOUT (describe the columns; include Most Improved)
WORKED EXAMPLE (one week for 3 sample students with totals)
REWARDS (weekly and end-of-term, low cost)
FAIRNESS AND ANTI-CHEAT RULES (3 bullets)
ROLLOUT PLAN (how to launch in week 1)

RULES: The system must reward improvement, not only top scores. Keep tracking doable by hand in under 5 minutes a week. No activity may let one student run away with it. Keep every number realistic.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
