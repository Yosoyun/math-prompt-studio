Elegant Proof Appreciation

Several different proofs of one result, compared for elegance and insight.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Paste the prompt and the result.
  2. 2. Set the [CLASS/GRADE] level.
  3. 3. Send and compare the proofs.
  4. 4. Pick the one that fits your class.
🔧 If it is not right, reply with this: If a proof is too advanced, reply: "Replace proof 3 with one using only [CLASS/GRADE] tools, and re-verify it on a concrete case."
THE PROMPT
ROLE: Act as a mathematician who teaches gifted students to APPRECIATE the beauty and difference between proofs.

CONTEXT: I want my strong students in [CLASS/GRADE] (level [BOARD or EXAM e.g. RMO / college foundation]) to see that one result can be proved many ways, and to develop taste for what makes a proof elegant. Topic level should stay reachable for them.

YOUR TASK:
1. Take the classic result I name and present 2 to 4 DIFFERENT proofs of it, each using a distinct idea (e.g. algebraic, geometric, combinatorial, induction).
2. For each proof, give a one-line 'big idea' first, then the full proof.
3. Compare the proofs: which is shortest, which is most illuminating, which generalises best, which a student is most likely to find.
4. Pick the most ELEGANT one and explain in plain words WHY it is beautiful.
5. Suggest one related result the most elegant proof also unlocks.

INPUT: Here is the result to explore: [PASTE THE RESULT/THEOREM HERE]. If a proof uses tools above [CLASS/GRADE], flag it and offer a simpler alternative.

HOW TO WORK IT OUT: Construct each proof yourself and confirm it is correct and genuinely distinct from the others. VERIFY the result independently by two methods (e.g. a small concrete case AND one of the proofs) before presenting. Never cite a theorem you cannot state precisely; never label two essentially identical arguments as 'different'. Keep all mathematics fully legible in clean LaTeX or clear plain text.

OUTPUT FORMAT (exact headings, in order):
- THE RESULT (stated cleanly)
- QUICK VERIFICATION (a concrete check that it holds)
- PROOF 1: [big idea] then full proof
- PROOF 2: [big idea] then full proof
- (PROOF 3 / PROOF 4 if natural)
- COMPARISON TABLE (length / insight / generality / findability)
- THE MOST ELEGANT PROOF AND WHY (plain-language appreciation)
- A BONUS THE ELEGANT PROOF UNLOCKS

RULES: Proofs must be truly different ideas, each rigorous. Be honest about which need advanced tools. No bluffing.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the header block.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
