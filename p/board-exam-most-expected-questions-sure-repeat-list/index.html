Board Exam Most-Expected Questions (Sure-Repeat List)

A ranked list of the most-likely board exam questions for a chapter, with model answers.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Open any AI chat.
  2. Fill in board, class and chapters.
  3. Paste the prompt and send.
  4. Share the list as a revision handout.
  5. Keep model answers for checking.
🔧 If it is not right, reply with this: If a listed question is off-syllabus or outdated, reply: "Question 5 is not in the current syllabus - replace it with a genuinely high-probability board question from the same chapter and give its model answer."
THE PROMPT
ROLE: Act as a senior board examiner and exam-trend analyst for school mathematics.

CONTEXT: I prepare [CLASS/GRADE e.g. Class 10 / Class 12] students for [BOARD e.g. CBSE / ICSE / State Board]. I want the questions MOST likely to appear in the board exam from specific chapters, so students revise the highest-yield material first.

YOUR TASK:
1. Produce a 'Most-Expected Questions' list of EXACTLY [NUMBER e.g. 20] questions from these chapters: [LIST CHAPTERS].
2. Rank them by likelihood: High / Medium probability, based on typical board patterns and frequently repeated question types (state your reasoning briefly per item).
3. For each question, give the marks it usually carries and a complete model answer written the way an examiner expects.
4. Highlight 5 'sure-shot' must-revise questions at the top.

INPUT: Here are the chapters and the marks weightage if I know it: [PASTE CHAPTERS AND WEIGHTAGE HERE]. If number is blank, use 20. Stay strictly inside the current board syllabus for the class.

HOW TO WORK IT OUT: For each question, solve fully internally, then VERIFY the final answer by two independent checks (alternate method, re-substitution, or a sanity check). Do NOT claim a question 'came in [year]' unless it is a well-known recurring type; instead justify likelihood by question-type frequency. Never invent formulas or values. Keep all maths legible in clean LaTeX or clear plain text.

OUTPUT FORMAT, in this exact order:

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
A. Title block: 'Board: [BOARD]', 'Class: [CLASS/GRADE]', 'Most-Expected Questions', 'Compiled by Indrajeet Yadav'.
B. SECTION 1 - TOP 5 SURE-SHOT: the five highest-probability questions with a one-line reason each.
C. SECTION 2 - FULL LIST: a table with columns Question No. | Chapter | Marks | Probability (High/Medium) | Why likely.
D. SECTION 3 - MODEL ANSWERS: examiner-style full solution for every question, with the two verification checks noted.
E. SECTION 4 - REVISION ORDER: the exact order students should revise these in.

RULES: Stay inside the current syllabus. No invented past-paper claims. Every model answer survives two checks. Mark scheme realistic for the board.

At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block.
