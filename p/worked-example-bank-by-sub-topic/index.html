Worked-Example Bank by Sub-Topic

A categorised bank of solved examples covering every sub-topic.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. List the chapter's sub-topics.
  2. 2. State class, board and examples per sub-topic.
  3. 3. Send the prompt.
  4. 4. Verify one example per sub-topic.
🔧 If it is not right, reply with this: If a category is thin, reply: "Add two more examples to the 'word problems' sub-topic at a harder level and verify each answer twice."
THE PROMPT
ROLE: Act as a mathematics author building a categorised bank of solved examples.

CONTEXT: I am assembling a worked-example bank for a book chapter on [TOPIC] for [CLASS/GRADE] students of [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. I want examples grouped by sub-topic so I can drop them into the right place in the book.

YOUR TASK:
1. For each sub-topic I give, write [NUMBER e.g. 3] solved examples.
2. Within each sub-topic, order examples easy to hard.
3. Solve each with one step per line and a short reason per key step.
4. End every example with a boxed final answer.
5. Tag each example with its sub-topic and difficulty (E/M/H).

INPUT: Here are the sub-topics to cover: [LIST SUB-TOPICS, e.g. factorisation method; quadratic formula; word problems; nature of roots].

HOW TO WORK IT OUT: Solve every example yourself before listing it; include only examples you can solve correctly. For each final answer, VERIFY by two independent checks (substitute back AND an alternate method or estimate). Never invent formulas or values. Keep all mathematics in clean LaTeX or clear plain text.

OUTPUT FORMAT (group by sub-topic, in this order per group):
1. Sub-topic Heading
2. Examples (numbered) each as: question, stepwise solution, boxed answer, tag [Sub-topic | E/M/H]

RULES: Clear grouping by sub-topic. Genuine difficulty progression. No skipped steps. No unsolved or wrong examples.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.

SIGNATURE: End your ENTIRE response with exactly this footer block (keep the lines and symbols):
