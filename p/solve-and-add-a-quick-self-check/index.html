Solve and Add a Quick Self-Check

A full solution plus a short, repeatable check students can do on any similar problem.

Attach a photo of the question first  ·  Best tool:  ChatGPT or Gemini (image)

✅ How to use this effectively

  1. 1. Photograph the question clearly.
  2. 2. Attach the photo to the chat.
  3. 3. Paste this prompt and send.
  4. 4. Read the solution and the quick check.
  5. 5. Try the check on a similar problem.
🔧 If it is not right, reply with this: If the check is unclear, reply: "Show the quick check worked out with real numbers from this exact problem, step by step."
THE PROMPT
ROLE: Act as a mathematics teacher who not only solves a problem but teaches students a fast way to check their own answer.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET], topic [TOPIC]. My students lose marks because they never check their work. I want a solution plus a simple self-check they can reuse.

YOUR TASK:
1. Read the question in the attached photo and restate it in clean text.
2. Solve it fully, step by step.
3. Then teach a QUICK self-check method suited to this type of problem (for example substituting back, dimensional/units check, estimating the size of the answer, or checking a special case).
4. Demonstrate that quick check using the actual numbers from this problem.

INPUT: I have attached a photo of the problem. If any part is unclear, tell me which symbol or line you cannot read, state your best assumption, and continue.

HOW TO WORK IT OUT: Solve the problem silently first. Then VERIFY the final answer with TWO independent checks. One of those checks must be the same quick self-check you will teach the student, shown fully with this problem's numbers. If the checks disagree, fix the error before replying. Never invent formulas or values; name standard results. Keep all mathematics fully legible in clean LaTeX or clear plain text, one step per line.

OUTPUT FORMAT (use these exact headings):
1. Question as I read it.
2. Step-by-step solution - numbered, one idea per line, short reasons.
3. Final answer - clearly stated.
4. The quick self-check - name the method in one line, then show it worked out with this problem's numbers.
5. Second verification - one more independent check in a line or two.
6. When this check works - one line on which problem types it suits.

RULES: The quick check must be genuinely fast - something a student can do in under a minute in an exam. Use plain ASCII only with straight quotes. Be exact unless an estimate is requested.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block at the top.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
