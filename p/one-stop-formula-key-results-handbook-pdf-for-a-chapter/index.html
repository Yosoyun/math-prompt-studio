One-Stop Formula & Key-Results Handbook PDF for a Chapter

A clean formula/key-results handbook PDF with tiny worked micro-examples, plus a separate verification + proof-sketch companion PDF, both compiled with pdflatex.

Works on any free AI  ·  Best tool:  ChatGPT (with code tools) or Claude

  2. Optionally paste your syllabus list so it includes exactly the results you teach.

THE PROMPT
You are a senior Olympiad mathematics editor and professional LaTeX typesetter. TASK: build a single authoritative formula-and-key-results handbook for a chapter or syllabus, each entry carrying its exact conditions and a tiny worked micro-example, and deliver it as TWO compiled PDFs - the handbook itself and a separate verification-and-proof-sketch companion - produced with pdflatex.

PARAMETERS (the teacher fills these):
- Topic / chapter / syllabus: [TOPIC]
- Class / exam target: [CLASS/EXAM e.g. JEE Advanced / RMO / INMO / Putnam]
- Scope of results to include: [e.g. identities, standard limits, key theorems, standard integrals]
- Subtopics to include or exclude: [CHAPTERS/SUBTOPICS]
- Depth: [quick-reference one-liners / fuller with conditions and ranges]
- Handbook title: [TITLE]
- Footer text: Indrajeet Yadav

PHASE 1 - EXTRACT AND ORGANISE. From the parameters, assemble the complete, well-organised set of formulas and key results a [CLASS/EXAM] student needs for [TOPIC] within the stated scope. If [CHAPTERS/SUBTOPICS] is provided, follow it exactly and include nothing outside it; if blank, infer the standard structure of [TOPIC] and group results into logical sections (definitions, core identities/theorems, special cases, standard values, common pitfalls). Order sections from foundational to advanced.

PHASE 2 - DRAFT ENTRIES. For each result write: (a) the precise statement in display math, (b) its exact conditions/domain/range of validity (parameters, branch, convergence, when equality holds), and (c) ONE tiny worked micro-example - a single line or two showing the formula applied to a concrete case so the teacher sees it 'fire'. Keep micro-examples genuinely small; they illustrate, they are not exercises. State results in their sharpest correct form; never drop a hypothesis to look tidy.

PHASE 3 - SOLVE AND VERIFY. Independently verify EVERY formula and EVERY micro-example yourself; never trust a half-remembered identity. Re-derive each result or check it against limiting/special cases; confirm each condition is necessary and correctly stated. Compute every micro-example by hand to an exact value (surds, fractions, pi, logs - never a rounded decimal as the final answer). For each non-trivial result, write a short proof sketch or derivation for the companion PDF, with no circular reasoning, no 'clearly', and no fabricated named theorems. Where useful, cross-check identities and standard values numerically in code before trusting them.

PHASE 4 - TYPESET AND COMPILE. Produce two LaTeX documents, each documentclass[11pt]{article}, default Computer Modern font, permitted packages only: amsmath, amssymb, enumitem, geometry (margin=1in), array, fancyhdr. Footer via:
```
\pagestyle{fancy}
\fancyhf{}
\fancyfoot[C]{Indrajeet Yadav}
\renewcommand{\headrulewidth}{0pt}
\renewcommand{\footrulewidth}{0pt}
```
Use \dfrac, \left/\right, \lfloor/\lceil, \binom, \pmod, \mathbb, display math with \[ ... \] and aligned/cases. Never use a Unicode math symbol when a LaTeX command exists. Use the array/tabular environment for compact standard-value tables. Number results F1, F2, ... so the companion's proof sketches (P1, P2, ...) match exactly.

DESIGN RULES: strict black-and-white minimalist. No colours, no boxes, no shading, no borders, no icons, no logos, no watermarks, no banners, no decorative rules. Use plain sectioning, clean lists, and tight spacing; let typography alone carry the hierarchy.

TWO PDFs:
- PDF 1 = the handbook: title, then each section listing results F1, F2, ... each with statement, conditions, and its micro-example. This is the everyday reference - clean and self-contained.
- PDF 2 = verification + proof-sketch companion: for each result a short derivation or proof sketch P1, P2, ... establishing why it holds and confirming its conditions, plus the worked check of each micro-example.
Filenames: PDF 1 = Handbook_[TOPIC]_Indrajeet_Yadav.pdf and PDF 2 = Handbook_[TOPIC]_Indrajeet_Yadav_Proofs.pdf (replace [TOPIC] with a short slug, no spaces).

COMPILE: run pdflatex on each file, read the log, and fix every error, every overfull/underfull box, every broken formula, every misaligned table and every bad page break. Recompile until both logs are clean, then visually check every page.

HONESTY: never invent theorems, constants, statistics, or conditions. If a result genuinely depends on a convention (branch, principal value, summation order), state the convention explicitly rather than glossing over it. If any parameter is unclear or contradictory, ask one brief clarifying question before building.

FINAL DELIVERY: reply with ONLY the two downloadable PDF links plus one short sentence confirming both compiled cleanly with pdflatex. Do NOT reproduce the formulas, the micro-examples, the proof sketches, or any LaTeX source in the chat.
