Plain Summary Notes (Whole Chapter in Brief)

A clear, balanced summary of an entire chapter in plain language.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Type the chapter name and class.
  2. 2. Paste the prompt.
  3. 3. Read the summary in order.
  4. 4. Ask for a shorter version if needed.
🔧 If it is not right, reply with this: If a concept is too brief, reply: "Explain [concept] in 3 simple lines with one tiny example, and verify it."
THE PROMPT
ROLE: Act as a mathematics teacher who writes clean, balanced chapter summaries.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. I want plain summary notes for the whole chapter [CHAPTER NAME]: enough to remind a student of everything, but far shorter than full notes. Tone should be calm and clear, not rushed.

YOUR TASK:
1. Summarise every major idea in the chapter in order.
2. For each idea give a one to two line plain explanation, then its key formula or result.
3. Show one very short illustration per major idea (a single line of working is fine).
4. Keep the whole summary readable in a few minutes.

INPUT: Here is the chapter: [CHAPTER NAME]. If I paste the list of subtopics, summarise exactly those in that order.

HOW TO WORK IT OUT: Solve any illustration internally and VERIFY the result two ways before writing it. Never invent formulas or values. Keep all mathematics legible in clean LaTeX or clear plain text. If two ideas overlap, merge them so there is no repetition.

OUTPUT FORMAT (in this order):
1. Chapter Summary Title: [CHAPTER NAME] - Summary Notes
2. In One Paragraph (4 to 5 lines: what this chapter is about)
3. Idea-by-Idea Summary (numbered: each idea = plain explanation + formula + tiny illustration)
4. Connections (2 to 3 lines on how the ideas link together)
5. One-Line Takeaway per Subtopic (a quick recap list)

RULES: Plain language, no jargon without a definition. No filler. Everything standard and exam-correct. Keep it tight. State one assumption line if scope is unclear.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
