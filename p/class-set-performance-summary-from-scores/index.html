Class-Set Performance Summary from Scores

A whole-class diagnosis: weak topics, common errors and what to reteach.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Paste the prompt below.
  3. 3. Paste your question-wise marks table.
  4. 4. Send and read the class diagnosis.
  5. 5. Use the action plan in your next class.
🔧 If it is not right, reply with this: If the analysis is too shallow, reply: "Go deeper: for the two weakest questions, name the likely misconception and give me one re-teaching example each, then recheck the totals."
THE PROMPT
ROLE: Act as a data-driven mathematics head of department who turns marks into a teaching plan.

CONTEXT: I teach Class [CLASS/GRADE] for [BOARD or EXAM]. A test is marked and I want a summary of how the whole class did, where they struggled, and what I should reteach - not individual remarks, but a class-level diagnosis.

YOUR TASK:
1. Read my question-wise marks data.
2. Identify the questions or topics where the class scored worst (lowest average percentage).
3. For each weak area, state the most likely underlying misconception.
4. Find any pattern (for example careless sign errors, or a specific skill missing).
5. Give a short, concrete reteaching action plan for the next two lessons.
6. Note the topics the class has clearly mastered so I do not waste time on them.

INPUT: Total marks: [TOTAL]. Topic of each question if known: [OPTIONAL MAP e.g. Q1 algebra, Q2 geometry]. Here is the data, one row per student with marks per question: [PASTE YOUR TABLE, e.g. "Name, Q1, Q2, Q3, ... Total"].

HOW TO WORK IT OUT: Compute the per-question averages internally first. VERIFY by two checks: (a) recompute the class average from the question averages and confirm it matches the average of the totals, and (b) re-rank the weakest questions to confirm the order. Never invent data. Keep numbers clean and rounded sensibly.

OUTPUT FORMAT, in this exact order:
1. TITLE BLOCK with test name, class, and the line "Compiled by Indrajeet Yadav".
2. SNAPSHOT: class average percentage, highest and lowest scoring questions.
3. PER-QUESTION TABLE: Question | Topic | Average percent | Status (Strong / OK / Weak).
4. LIKELY MISCONCEPTIONS for the weakest 2 to 3 questions.
5. RETEACHING ACTION PLAN: a numbered 2-lesson plan with one worked example idea each.
6. MASTERED TOPICS (one line list).
7. VERIFICATION NOTE confirming the two checks.

RULES: Stay at class level - do not single out individual children by name. Be specific about misconceptions, not vague. If my data is incomplete, say what is missing rather than guessing.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
