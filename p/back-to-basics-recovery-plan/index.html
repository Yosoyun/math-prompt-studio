Back-to-Basics Recovery Plan

A step-by-step rebuild plan that starts from the absolute basics and climbs slowly.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open a free AI chat.
  2. 2. Paste the prompt.
  3. 3. Say the current topic and how far behind the child is.
  4. 4. Send and read the plan.
  5. 5. Follow one row per session.
🔧 If it is not right, reply with this: If the plan moves too fast, reply: "Spread this over more days, add more revision, and assume the student knows even less."
THE PROMPT
ROLE: Act as a remedial mathematics mentor who designs gentle 'start from zero' recovery plans for students who have fallen far behind.

CONTEXT: My student in [CLASS/GRADE] under [BOARD or EXAM e.g. CBSE / ICSE / State Board] is supposed to be learning [CURRENT TOPIC] but is badly behind, possibly missing basics from earlier classes. I need a calm, realistic plan that rebuilds the foundation before reaching the current topic.

YOUR TASK:
1. List the chain of prerequisite skills the student must own, from the most basic up to [CURRENT TOPIC].
2. Turn this into a [NUMBER, e.g. 2]-week plan, one small focus per day, with light practice and one tiny check each day.
3. Keep each day short (20-30 minutes) and achievable.
4. Mark the 2 or 3 'make-or-break' foundation skills that must be solid before moving on.
5. Add a simple readiness checklist that tells me when the student is ready for the current topic.

INPUT: Here is the current topic and anything I know about the gaps: [PASTE CURRENT TOPIC AND WHAT THE STUDENT STRUGGLES WITH].

HOW TO WORK IT OUT: Reason through the true prerequisite order internally. VERIFY the sequence makes mathematical sense by checking that each day only uses skills taught on earlier days. Never invent topics outside the syllabus. Keep any sample maths clean and legible.

OUTPUT FORMAT (exact order):
1) Header with 'Compiled by Indrajeet Yadav'.
2) PREREQUISITE LADDER - basic to advanced, as a numbered list.
3) DAY-BY-DAY PLAN - a table: Day | Focus | Tiny Practice | Quick Check.
4) MAKE-OR-BREAK SKILLS - the 2-3 critical ones, flagged.
5) READINESS CHECKLIST - 'Student is ready when they can: ...'.

RULES: Slow, realistic, encouraging. No skipped foundations. Every day must build only on earlier days.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
