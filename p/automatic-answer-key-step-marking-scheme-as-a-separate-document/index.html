Automatic Answer Key + Step-Marking Scheme as a Separate Document

A complete examiner's answer key with step-wise marks for every part, ready as its own PDF.

Works on any free AI  ·  Best tool:  Claude (free) or ChatGPT (free), paste into Word for PDF

✅ How to use this effectively

  1. 1. Paste the EXACT paper you set.
  2. 2. State total marks for cross-check.
  3. 3. Read the verification note it returns.
  4. 4. Copy into Word, export as PDF.
  5. 5. Keep this file separate from the paper.
🔧 If it is not right, reply with this: If a step breakdown is vague, type: 'For Question [n], split the marks into specific awardable steps (formula = X marks, substitution = X, final answer = X) and show the exact value at each step.'
THE PROMPT
ROLE: Act as a chief examiner who writes official step-marking schemes (marking rubrics) for board mathematics papers.

CONTEXT: I teach [CLASS/GRADE] [SUBJECT] for [BOARD]. I am giving you a complete question paper. I need a SEPARATE answer-key document with full solutions and a step-wise marking scheme an assistant examiner could apply consistently.

YOUR TASK:
1. For every question and sub-part, give the complete worked solution.
2. Break each solution into awardable steps and assign marks to each step so the parts sum to that question's marks (e.g. correct formula = 1, correct substitution = 1, correct simplification = 1, final answer with unit = 1).
3. State the final answer clearly on its own line, with units where relevant.
4. Add 'Accept also:' notes for valid alternative methods or equivalent forms (e.g. accept 0.5 for 1/2).
5. Add 'Common error / no marks for:' notes flagging typical mistakes (e.g. sign error, missing unit).
6. End with a verification line per section and a grand-total check.

INPUT: Here is the full question paper: [PASTE THE ENTIRE PAPER]. Stated total marks = [TOTAL MARKS].

HOW TO WORK IT OUT: Solve each question independently and completely first. VERIFY every final answer by two independent methods where possible (e.g. solve algebraically AND check by substitution; or compute AND estimate to confirm reasonableness). Re-add the step marks for each question to equal its stated marks, then re-add all questions to equal [TOTAL MARKS]. Never invent values not derivable from the question. Keep mathematics legible in plain text: fractions a/b, powers x^2, roots sqrt(x), and show every intermediate line.

OUTPUT FORMAT (exact order, per question):
- Q[number]: Solution (numbered steps with marks in brackets per step).
- Final Answer: [value + unit].
- Accept also: [...].
- No marks for: [...].
After all questions: a 'Marks Reconciliation' block listing each section total and the grand total, ending with 'Grand total verified = [TOTAL MARKS]'.

RULES: Marks per question MUST match the paper exactly. If any question in the paper is ambiguous or unsolvable as written, flag it under a 'Setter Alerts' list at the top with a suggested fix instead of guessing. Keep formatting plain so it pastes cleanly into Word and exports as a PDF.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the document title block (e.g. 'MARKING SCHEME - Compiled by Indrajeet Yadav').
