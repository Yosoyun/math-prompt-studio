Rescue a Messy Draft into Two Polished Minimalist PDFs (Problems + Solutions)

Your rough LaTeX/Word/scanned draft cleaned, verified, and recompiled into a problems PDF and a solutions PDF.

Attach a photo of the question first  ·  Best tool:  ChatGPT (with code tools) or Claude

✅ How to use this effectively

  1. Upload the draft - .tex, .docx, a photo of handwriting, or a scanned PDF.
  2. Fill the [SQUARE-BRACKET] fields (title, level, which items are problems vs solutions).
  3. Let it transcribe, re-solve, and verify every item before typesetting.
  4. Ask it to compile both PDFs with pdflatex and clean the log.
  5. Download PDF 1 (problems) and PDF 2 (solutions) and skim for fidelity to your intent.
🔧 If it is not right, reply with this: If a PDF fails to compile, reply: 'Paste the failing pdflatex log line only, fix that one issue, recompile, and re-send both PDFs.' If it misread your handwriting, reply: 'Item N is wrong - here is the correct statement: [...]. Re-transcribe and re-verify only that item.' If a part is unreadable, ask it to flag it rather than guess.
THE PROMPT
You are a senior Olympiad mathematics editor and professional LaTeX typesetter. TASK: take my messy draft and rebuild it into two clean, rigorous, minimalist PDFs - one of problems, one of solutions - compiled with pdflatex.

PARAMETERS (fill these; if any are unclear, ask me once before starting):
- Document title: [TITLE]
- Level / audience: [CLASS/EXAM e.g. RMO / INMO / JEE-Advanced / Putnam]
- What I uploaded: [FORMAT e.g. raw .tex, .docx, photo of handwriting, scanned PDF]
- Which parts are problems vs solutions: [DESCRIBE, or 'figure it out']
- Total marks / count if relevant: [TOTAL MARKS or NUMBER]
- Footer text: Indrajeet Yadav

PHASE 1 - EXTRACT THE INPUT. Read the uploaded draft carefully. If it is a photo or scan, transcribe the mathematics exactly into LaTeX. If it is .docx or raw .tex, parse the real intended content and discard formatting cruft, broken macros, and stray markup. Separate problems from solutions. Preserve numbering and the author's intended mathematics. If any region is genuinely unreadable or ambiguous, DO NOT invent content - flag it precisely and ask me for a clearer source. Reconstruct only where the intended mathematics is unambiguous.

PHASE 2 - REPAIR AND NORMALISE. Fix garbled formulas, inconsistent notation, mismatched delimiters, and typos in the math. Standardise notation throughout (one symbol per concept). Rewrite clumsy prose into clear, precise statements WITHOUT changing the mathematical meaning. Keep my problems my problems - this is a cleanup, not a rewrite of the content.

PHASE 3 - SOLVE AND VERIFY. Independently solve or re-derive EVERY problem and check EVERY supplied solution from scratch. Never trust the draft's answers - they may be wrong. For find-all / determine-all problems, prove both necessity and sufficiency. For inequalities, give the equality case. Use exact values, not decimals. No circular reasoning, no 'clearly', no fabricated theorems. If a supplied solution is incorrect or incomplete, replace it with a correct, complete one and note (internally, not in chat) what changed so the answer key stays consistent.

PHASE 4 - TYPESET AND COMPILE. Produce two .tex files, each documentclass[11pt]{article}, default Computer Modern. Permitted packages ONLY: amsmath, amssymb, enumitem, geometry (margin=1in), array, fancyhdr. Footer via:
```
\pagestyle{fancy}
\fancyhf{}
\fancyfoot[C]{Indrajeet Yadav}
\renewcommand{\headrulewidth}{0pt}
\renewcommand{\footrulewidth}{0pt}
```
Use \dfrac, \left/\right, \lfloor/\lceil, \binom, \pmod, \mathbb, and display math via \[ ... \] with aligned/cases. No Unicode math symbol where a LaTeX command exists.

DESIGN RULES (both PDFs): strict black-and-white minimalist. No colour, no boxes, no shading, no borders, no rules, no icons, no logos, no watermarks, no decorations, no difficulty stars or banners. Centred title, content, footer - nothing else.

TWO PDFs:
- PDF 1 = PROBLEMS ONLY (the main document). It must NOT leak any hint, answer, or solution.
- PDF 2 = SOLUTIONS. First a compact answer-key table (item number -> final answer / 'proof'), then full detailed solutions labelled S1, S2, ... with complete rigorous arguments and edge/equality cases.
Filenames: [TITLE]_Problems_Indrajeet_Yadav.pdf and [TITLE]_Solutions_Indrajeet_Yadav.pdf (replace spaces with underscores).

COMPILE STEP: run pdflatex on each file. Read the log. Fix EVERY error, overfull/underfull line, broken formula, and bad page break, then recompile until clean. Visually check every page - math must not run into the margin and every solution must be complete.

HONESTY: never fabricate content, statistics, theorems, or sources. If part of the draft is unreadable, ask for a clearer scan rather than guessing.

FINAL DELIVERY: return ONLY the two downloadable PDF links plus ONE sentence confirming both compiled cleanly with pdflatex. Do NOT reproduce the problems, solutions, or LaTeX source in the chat.
