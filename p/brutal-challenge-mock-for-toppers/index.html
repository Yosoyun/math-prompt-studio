Brutal Challenge Mock for Toppers

A hard, top-tier mock to stretch your strongest students, with full solutions.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. List chapters and total marks.
  2. 2. Name the target (e.g. JEE Advanced).
  3. 3. Send the prompt.
  4. 4. Ask for hints if students get stuck.
  5. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If a question is impossibly hard or broken, reply: "Question 6 may be unsolvable at this level - confirm it has a clean answer or replace it with a fair-but-hard question."
THE PROMPT
ROLE: Act as an elite competition paper-setter who designs hard-but-fair challenge papers.

CONTEXT: I teach a top batch in [CLASS/GRADE] preparing for [BOARD or EXAM e.g. JEE Advanced / Olympiad / KVPY]. My strongest students are bored by routine papers. I want a genuinely challenging mock that still has clean, correct answers.

YOUR TASK:
1. Build a hard mock from these chapters: [LIST CHAPTERS]. Total marks: [TOTAL MARKS]. Time: [TIME].
2. Emphasise multi-step reasoning, clever combinations of concepts, and well-known traps - but every question MUST have a clean, reachable answer. Hard, not impossible or unfair.
3. For each question add a 'Difficulty' rating (Hard / Very Hard) and a one-line HINT (kept in a separate hints section, not on the student paper).
4. Provide complete, elegant solutions and an answer key.

INPUT: Here are the chapters and marks: [PASTE DETAILS HERE].

HOW TO WORK IT OUT: Solve every question fully internally first to PROVE it is solvable at this level. VERIFY each final answer two independent ways. Never invent formulas or values, and never make a question whose answer you cannot reach cleanly. Keep all maths legible in clean plain text or LaTeX.

OUTPUT FORMAT (exact order):
STUDENT PAPER: header, numbered questions (no hints shown), each tagged Hard or Very Hard.
--- HINTS (separate page) ---: one short hint per question.
SOLUTIONS: full elegant steps, final answer labelled.
ANSWER KEY: compact list.
VERIFICATION NOTE: confirm every question is solvable and answers double-checked.

RULES: Hard but fair - no broken or unsolvable questions. Marks total [TOTAL MARKS]. Original questions. Calibrated to [BOARD or EXAM] top level.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
