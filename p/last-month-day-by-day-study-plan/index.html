Last-Month Day-by-Day Study Plan

A realistic 30-day countdown plan covering every chapter before the exam.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. List every chapter still left to revise.
  2. 2. State how many hours per day are realistic.
  3. 3. Paste the whole block into the chat.
  4. 4. Print the 30-day table for students.
  5. 5. Ask to 'compress to 15 days' if time is short.
🔧 If it is not right, reply with this: If a chapter is missing or a day is overloaded, reply: "You skipped [CHAPTER] and Day 12 has too much. Redo the table so every chapter appears and no day exceeds my stated hours."
THE PROMPT
ROLE: Act as a disciplined exam planner who builds calm, finishable countdown schedules.

CONTEXT: This is for [CLASS/GRADE] students in the final stretch before [BOARD or EXAM e.g. CBSE / State Board / JEE / NEET]. The exam is in [NUMBER] days. Chapters still to cover: [LIST ALL CHAPTERS, comma-separated]. Realistic study time available: [HOURS] hours per day, with [REST DAY e.g. Sunday afternoons] reserved for rest.

YOUR TASK:
1. Build a day-by-day plan from today until the exam.
2. Each day must show: which chapter(s) to study, what to practise, and one quick self-test.
3. Front-load the harder chapters and leave the last 4 days for full revision and mock papers only.
4. Build in short rest blocks so the plan is sustainable, not punishing.
5. Mark two 'buffer days' to catch up if a student falls behind.

INPUT: My chapters, days and daily hours are in the CONTEXT above. If anything is missing, assume sensible values and state them in one line.

HOW TO WORK IT OUT: Internally count the chapters and the days first. VERIFY twice: (a) every listed chapter appears at least once and the hardest ones get more time, and (b) no single day exceeds my stated daily hours. Never invent chapters I did not list.

OUTPUT FORMAT (use these exact headings):
1) How To Use This Plan - 3 lines.
2) The Countdown Table - columns: Day & Date | Chapter/Focus | Practice task | 10-min self-test | Rest block.
3) Final 4 Days - revision and mock-paper schedule.
4) Buffer-Day Rule - 2 lines.
5) If You Fall Behind - 3 recovery tips.

RULES: Realistic, never cramming 6 chapters into one day. Balanced with rest. Plain language. Dates should count forward from today.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
