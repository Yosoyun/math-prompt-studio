Revision Web of a Whole Unit

A revision web linking several chapters of one unit into one connected map.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. List all chapters in the unit.
  3. 3. Paste the prompt below.
  4. 4. Send and review the connections.
  5. 5. Ask it to add cross-links you noticed.
🔧 If it is not right, reply with this: If links between chapters are missing, reply: "Add a clear CONNECTIONS section showing how each chapter feeds into the next, with one line per link." If something looks wrong, reply: "Re-check that connection and explain why it holds."
THE PROMPT
ROLE: Act as a senior mathematics teacher who builds revision webs that connect every chapter inside a unit.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. Just before exams I want a single REVISION WEB (text form) for one full unit, so students see how the chapters connect, not as isolated islands. No image needed.

YOUR TASK:
1. Treat the unit name as the centre.
2. Make one branch per chapter in the unit.
3. Under each chapter list its 3 to 5 must-know points (key idea, main formula, one typical question).
4. Then add a CONNECTIONS section that explicitly links chapters to each other (what is a prerequisite for what, what builds on what, what is commonly combined in mixed questions).

INPUT: Here is the unit and the chapters it contains: [PASTE UNIT NAME AND LIST OF CHAPTERS HERE]. If I list only the unit, use the standard [BOARD] chapter breakdown for [CLASS/GRADE].

HOW TO WORK IT OUT: First map all chapters privately, then trace the genuine logical links between them. VERIFY each formula and each claimed connection twice - once for syllabus accuracy and once by checking it makes mathematical sense. Never invent links or formulas; if a link is doubtful, mark it "(verify)". Keep all maths in clean plain text (x^2, sqrt(x), sin x).

OUTPUT FORMAT (exactly this order):
UNIT: <unit name>
Chapter A - <name>
   - must-know point
   - must-know point
Chapter B - <name>
   - must-know point
(continue for all chapters)
CONNECTIONS (how the chapters link):
   - <Chapter A> is needed before <Chapter B> because ...
   - <Chapter C> and <Chapter D> are combined in mixed problems like ...
END-OF-UNIT QUICK CHECK: 5 one-line questions that touch multiple chapters.

RULES: Be accurate and concise. Show real connections, not vague ones. Match depth to [CLASS/GRADE]. Fit on two pages.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.

SIGNATURE: End your ENTIRE response with exactly this footer block (keep the lines and symbols):
