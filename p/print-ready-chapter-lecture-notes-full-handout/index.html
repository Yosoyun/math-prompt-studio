Print-Ready Chapter Lecture Notes (Full Handout)

A complete, print-ready chapter handout: theory, derivations, worked examples, traps and a recap, ready for Word/PDF.

Works on any free AI  ·  Best tool:  Claude or ChatGPT (free)

✅ How to use this effectively

  1. Open Claude or ChatGPT.
  2. Fill [CHAPTER], [CLASS] and [BOARD].
  3. Paste the prompt and send.
  4. Type 'continue with the next section' as needed.
  5. Paste into Word, add page breaks, export PDF.
🔧 If it is not right, reply with this: If a derivation looks shaky, reply: 'Re-derive [step] from first principles, justify each line, and re-verify the final result by a worked numerical example.'
THE PROMPT
ROLE: Act as a senior mathematics author writing publication-quality lecture notes for a printed student handout.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / JEE / NEET]. Prepare a complete, self-contained set of lecture notes for the chapter [CHAPTER NAME]. Target reading level: a sincere student who missed the class and must learn it from these notes alone.

YOUR TASK:
1. Write the full chapter as a printable handout: clear theory, every key formula with a one-line 'why', and short derivations of the main results.
2. Include 6-10 graded worked examples (easy to hard), each with the full method, the final boxed answer, and a 'why this works' remark.
3. Add a 'Common Traps' box (5 items) and a 'Speed Tricks / shortcuts' box where legitimate.
4. Provide a one-page formula sheet at the end and a 6-line chapter recap.
5. Insert clearly labelled placeholders like [DIAGRAM: right triangle with sides a,b,c] wherever a figure helps, so I can sketch it in.

INPUT: Here is the exact syllabus scope to cover (sub-topics, depth, anything to exclude): [PASTE SCOPE]. If I attach my textbook pages, follow their notation and sequence.

HOW TO WORK IT OUT: Derive results properly and VERIFY every worked example by a second independent method (substitute back, special case, or alternate route). Never invent formulas, constants, or values; if something is outside standard syllabus, mark it 'enrichment'. Keep all mathematics fully legible in plain ASCII (^ for powers, sqrt(), a/b for fractions, <= and >= for inequalities). Re-check each final answer before printing it.

OUTPUT FORMAT (in this order, with headings):

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
1. TITLE BLOCK: Chapter, Class, Board, 'Compiled by Indrajeet Yadav'.
2. LEARNING MAP: 4-6 bullet outcomes.
3. THEORY with numbered sub-sections and key-formula boxes.
4. DERIVATIONS (short, justified).
5. WORKED EXAMPLES (numbered, graded, boxed answers).
6. COMMON TRAPS box.
7. SHORTCUTS box (only legitimate ones).
8. ONE-PAGE FORMULA SHEET.
9. CHAPTER RECAP (6 lines).

RULES: Self-contained and printable. Plain ASCII maths only. Number everything for easy reference. Keep tone clear and exam-focused, not chatty.

At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block at the top.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
