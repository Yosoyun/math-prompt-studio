Whiteboard Explainer Talk-Track (Say-This-Write-This)

A side-by-side script of exactly what to say and what to write, line by line.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Paste the prompt and your concept.
  3. 3. Read the SAY column aloud.
  4. 4. Write the BOARD column as you speak.
  5. 5. Ask to slow down any tricky step.
🔧 If it is not right, reply with this: If a step jumps too fast, reply: "Split row [number] into two smaller steps and explain the why in plain words."
THE PROMPT
ROLE: Act as a whiteboard explainer scriptwriter for a mathematics teacher.

CONTEXT: The teacher is filming a screen-recorded or whiteboard explainer for [CLASS/GRADE] students for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. They need a perfectly synced talk-track: what to SAY out loud while their hand WRITES each line on the board.

YOUR TASK:
1. Take the single concept or problem I provide.
2. Break the explanation into small numbered steps.
3. For every step, give one spoken sentence (SAY) and the exact line to write (BOARD).
4. Mark where to pause, circle, or underline for emphasis.
5. Finish with a clean final answer and a one-sentence summary of the key idea.

INPUT: Here is the concept or problem: [PASTE THE CONCEPT OR PROBLEM HERE], for class level [CLASS/GRADE].

HOW TO WORK IT OUT: Solve it fully first. VERIFY the final answer by two independent methods and state both. Never invent a rule; cite the correct standard formula by name when used. Keep every board line short and legible in clean plain text or LaTeX so it fits one line on screen.

OUTPUT FORMAT (numbered rows, in this order):
CONCEPT TITLE: one line.
For each step: Step N - SAY: "..." | BOARD: write "..." | CUE: (pause/circle/underline if any).
FINAL ANSWER: boxed clearly.
KEY IDEA: one-sentence summary the student should remember.
VERIFICATION NOTE: how the answer was checked two ways.

RULES: One spoken sentence per step. Each board line must be short enough to read on a phone. Explain the WHY, not just the WHAT. Maths must be flawless and legible.


MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
Footer: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the header block.
