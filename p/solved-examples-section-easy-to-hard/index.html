Solved-Examples Section (Easy to Hard)

A ladder of fully worked examples that model the chapter's methods.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Enter topic, class, board and example count.
  2. 2. Send the prompt.
  3. 3. Verify the trickiest example yourself.
  4. 4. Request a different sub-type if missing.
🔧 If it is not right, reply with this: If steps are skipped, reply: "Show every intermediate step for Example 5, explain the reason for each, and verify the answer a second way."
THE PROMPT
ROLE: Act as a mathematics author writing model solved examples for a textbook.

CONTEXT: I need a solved-examples section for a chapter on [TOPIC] for [CLASS/GRADE] students of [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. The examples should teach method, not just give answers, and should cover the chapter's main question types.

YOUR TASK:
1. Write [NUMBER e.g. 8] solved examples ordered from easy to hard.
2. Cover each major method or question type of the chapter at least once.
3. For each example, state the question, then solve it with one step per line.
4. Add a short reason beside or below each key step.
5. End each example with a clearly boxed final answer.
6. After each example add a one-line "Method Note" naming the technique used.

INPUT: Here are the sub-topics or specific methods to cover: [PASTE HERE OR WRITE "all standard types"].

HOW TO WORK IT OUT: Solve each example fully yourself before writing it; include only examples you can solve correctly. For each final answer, VERIFY by two independent checks (for example substitute back AND use an alternate method or estimate). Never invent formulas or values. Keep all mathematics in clean LaTeX or clear plain text.

OUTPUT FORMAT (exact order, repeat per example):
1. "Example n:" question statement
2. Solution (numbered steps with reasons)
3. Final Answer (boxed)
4. Method Note (one line)

RULES: No skipped steps. Genuine easy-to-hard progression. Every method labelled. No unsolved or wrong examples.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
