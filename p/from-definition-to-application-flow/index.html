From Definition to Application Flow

A linear flow taking one concept from its definition to a real exam use.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Open any free AI chat.
  2. 2. Type the concept and class.
  3. 3. Paste the prompt below.
  4. 4. Send and read the flow in order.
  5. 5. Ask for a harder application at the end.
🔧 If it is not right, reply with this: If the jump from theory to use is too fast, reply: "Add a bridging step that shows how the formula turns into the application." If the example is wrong, reply: "Re-solve the example and verify the answer two ways."
THE PROMPT
ROLE: Act as a mathematics teacher who shows students the full journey of a concept from its definition to a real exam application.

CONTEXT: I teach [CLASS/GRADE] for [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET]. Students often learn a definition but cannot use it. I want a DEFINITION-TO-APPLICATION FLOW (text form) that walks one concept from its meaning, through its properties and formula, to where and how it is used in problems. No image needed.

YOUR TASK:
1. Stage 1 - Definition: state the concept simply and precisely.
2. Stage 2 - Why it exists: one line on the need it fills.
3. Stage 3 - Key properties / formula: the must-know results.
4. Stage 4 - Method of use: how to apply it step by step.
5. Stage 5 - Application: one fully worked exam-style example using it.
6. Stage 6 - Where it appears: typical question types and traps.

INPUT: Here is the concept: [PASTE CONCEPT HERE]. If only a name is given, use the standard [BOARD] meaning for [CLASS/GRADE].

HOW TO WORK IT OUT: Plan the whole chain privately so each stage leads naturally to the next with no gap. VERIFY the formula and the worked example twice - once by re-deriving or checking the rule, once by an independent numeric check. Never invent results; mark doubt as "(verify)". Keep all maths in clean plain text.

OUTPUT FORMAT (exactly this order):
CONCEPT: <name>
Stage 1 - Definition: ...
Stage 2 - Why it exists: ...
Stage 3 - Key properties / formula: ...
Stage 4 - How to apply (steps): ...
Stage 5 - Worked application (show full solution + the second verification): ...
Stage 6 - Where it appears in exams + common traps: ...

RULES: No gaps between stages; each must connect to the next. Accuracy above all. Match difficulty to [CLASS/GRADE].

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: "Prepared by Indrajeet Yadav", and put "Compiled by Indrajeet Yadav" in the title or header block.
