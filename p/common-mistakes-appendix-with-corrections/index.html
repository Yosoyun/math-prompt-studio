"Common Mistakes" Appendix With Corrections

A wrong-vs-right appendix showing typical errors and the fix.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. List the chapters or topics.
  2. 2. State class and board.
  3. 3. Send the prompt.
  4. 4. Confirm each mistake is one you actually see.
🔧 If it is not right, reply with this: If an example is unrealistic, reply: "Replace mistake 4 with a more common real error students make, and show the correct working in full."
THE PROMPT
ROLE: Act as an experienced examiner cataloguing common student errors.

CONTEXT: I am writing a "Common Mistakes" appendix for a book for [CLASS/GRADE] students of [BOARD or EXAM e.g. CBSE / ICSE / State Board / JEE / NEET] covering [CHAPTER OR TOPIC LIST]. The aim is to show students the exact wrong working they tend to do, then the correct way.

YOUR TASK:
1. List [NUMBER e.g. 12] of the most common, realistic mistakes for these topics.
2. For each mistake, show: the Wrong Working, why it is wrong, and the Correct Working.
3. Add a one-line "How To Avoid It" tip for each.
4. Group mistakes by topic.
5. Order them from most common to less common.

INPUT: Here are the topics, and any specific errors I keep seeing: [PASTE HERE].

HOW TO WORK IT OUT: Recall genuinely common errors internally first; do not invent rare or fake ones. For each correct working, VERIFY the final result by two independent checks. Make sure the "why it is wrong" explanation is accurate and the corrected method is standard. Never invent formulas. Keep all mathematics in clean LaTeX or clear plain text, with wrong and correct lines clearly distinguished.

OUTPUT FORMAT (group by topic; per mistake in this order):
1. "Mistake n:" short description
2. Wrong Working (labelled)
3. Why It Is Wrong (one or two lines)
4. Correct Working (labelled, stepwise)
5. How To Avoid It (one line)

RULES: Only realistic, common mistakes. Wrong and correct working clearly labelled. Correct working fully verified. Grouped by topic, ordered by frequency.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
