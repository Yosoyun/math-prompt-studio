Real-Life Application Worksheet That Makes Maths Matter

A context-rich worksheet of real-world problems on one topic, with data, sub-parts, and a worked answer key.

Works on any free AI  ·  Best tool:  ChatGPT (free) or Claude (free)

✅ How to use this effectively

  1. 1. Set the topic and class.
  2. 2. Describe your students' daily context.
  3. 3. Ask for local, familiar scenarios.
  4. 4. Check the numbers are realistic.
  5. 5. Print with the answer key removed.
🔧 If it is not right, reply with this: If the contexts feel fake or forced, type: 'Use everyday Indian scenarios (auto fares, electricity bills, cricket scores, recipe scaling) with realistic 2025 numbers.'
THE PROMPT
Act as a mathematics teacher who specialises in authentic, real-world problem design that connects a topic to students' actual lives.

CONTEXT: Class [CLASS/GRADE], board/exam [BOARD or EXAM], topic [TOPIC, e.g. percentages and ratio]. My students live in [CONTEXT, e.g. an Indian semi-urban town], and I want a worksheet where every question is grounded in a believable real-life situation, so students see why the maths matters.

YOUR TASK:
1. Create 8 application problems on [TOPIC], each set in a concrete real-life scenario (shopping discounts, electricity or mobile bills, travel and fuel, recipes, sports stats, savings, area for tiling or painting, and similar).
2. Use realistic, current-feeling numbers and units; avoid contrived values.
3. Give the harder questions 2-3 linked sub-parts (a, b, c) that build on the same scenario.
4. Include at least 2 problems that require students to interpret or critique an answer in context (for example, 'Is this offer actually cheaper? Justify.').
5. Keep all questions on [TOPIC]; the context is the dressing, the maths is the point.

INPUT: Topic, class and student context are above. If I paste a real local detail here: [PASTE or leave blank], weave it in so the sheet feels familiar.

HOW TO WORK IT OUT: Solve each problem fully before writing it. VERIFY every answer by two independent checks and a sanity check that the result is realistic in context (no negative ages, no impossible discounts). Never invent formulas or unrealistic data; keep answers clean. Keep mathematics legible in plain text (^ for powers, sqrt(), / for fractions, clear units and currency).

OUTPUT FORMAT (exact order):
- Title block with 'Compiled by Indrajeet Yadav', topic, Name / Class / Date lines.
- Short note: 'Show working and units; round sensibly.'
- Questions 1-8 (with sub-parts where stated).
- '--- ANSWER KEY ---' with final answers, units, and a one-line interpretation for the 'justify/critique' questions.

RULES: Authentic, respectful contexts. Realistic numbers. Units and rounding handled properly. A4-friendly with space to work. No invented or absurd data. State one assumption if unclear and proceed.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
