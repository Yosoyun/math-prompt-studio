THE PROMPT
ROLE: Act as a senior mathematics examiner and board paper-setter.

CONTEXT: This paper is for Class [CLASS/GRADE], [BOARD or EXAM e.g. CBSE / ICSE / State Board], for a full examination. Total marks [TOTAL MARKS], time [TIME e.g. 3 hours]. The setter is a busy teacher who will print this directly, so it must be exam-accurate and complete.

YOUR TASK:
1. Build a blueprint (weightage) table first: rows are chapters, columns are the sections, cells show number of questions and marks; add a TOTAL row.
2. Write the full paper divided into clearly labelled sections, with marks shown beside every question.
3. Mix difficulty as [e.g. 40 percent easy, 40 percent moderate, 20 percent hard].
4. Write a complete answer key after the paper.

INPUT: Here are the chapters or topics to cover: [LIST CHAPTERS or TOPICS]. Use only these. If I leave a field blank, assume the standard latest pattern for this class and state that assumption in one line at the top.

HOW TO WORK IT OUT: Solve every question internally before printing it. Verify each final answer by at least two independent checks (re-substitution, an alternate method, or a sign and units sanity check). Never invent theorems, formulas or values. Keep all mathematics fully legible using clean LaTeX or clear plain text. Confirm that section marks sum exactly to the total; if they do not, fix the paper before showing it.

OUTPUT FORMAT, in this exact order:
1. HEADER BLOCK: school name line '__________', 'Class: [CLASS/GRADE]', 'Subject: Mathematics', 'Board: [BOARD or EXAM]', 'Time: [TIME]', 'Maximum Marks: [TOTAL MARKS]', and 'Compiled by Indrajeet Yadav'.
2. GENERAL INSTRUCTIONS: numbered, 5 to 7 lines.
3. BLUEPRINT TABLE with a TOTAL row that equals [TOTAL MARKS].
4. THE QUESTION PAPER: sections labelled, marks against every question.
5. ANSWER KEY: final answer to every question, plus a 1 to 3 line working outline for long questions.

RULES: Stay strictly inside the listed syllabus. No question may be unsolvable or ambiguous. Number every question. Marks must total exactly.

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

SIGNATURE: At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the header block.
