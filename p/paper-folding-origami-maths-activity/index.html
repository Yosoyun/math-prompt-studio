Paper-Folding (Origami Maths) Activity

A fold-by-fold activity that reveals a maths idea, with verification.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. 1. Type the concept and class in the brackets.
  2. 2. Paste the prompt into any AI chat.
  3. 3. Read the fold-by-fold steps.
  4. 4. Try the folds yourself once first.
  5. 5. Run it with the class.
🔧 If it is not right, reply with this: If a fold is unclear, reply: "Rewrite step [number] as smaller sub-steps a child can follow, and describe exactly which edges meet."
THE PROMPT
ROLE: Act as a maths teacher who uses paper folding (origami maths) to teach geometric ideas concretely.

CONTEXT: I teach [CLASS/GRADE] under [BOARD e.g. CBSE / ICSE / State Board]. I want a PAPER-FOLDING ACTIVITY that uses only a plain sheet (and maybe a ruler) to reveal or prove the concept: [CONCEPT e.g. a perpendicular bisector / angle bisector / that a triangle's angles sum to 180 / equal areas / fractions]. It must fit one period.

YOUR TASK:
1. State the concept the folding will reveal.
2. Specify the paper needed (size/shape).
3. Give clear fold-by-fold steps; for each fold, say exactly which edges or points meet.
4. Explain what each completed fold shows mathematically.
5. Provide one short worked example tying the folds to numbers or a property.
6. Add 2 check questions with answers.

INPUT: Here is my concept and paper available: [PASTE CONCEPT AND PAPER TYPE HERE]. If blank, choose a classic folding activity for the class and board above.

HOW TO WORK IT OUT: Mentally perform every fold to be sure the steps actually work and describe edge-meetings precisely. VERIFY the resulting property two ways (for example, by the folding logic and by a calculation) and show both checks. Never claim a result the folds do not produce. Keep all maths legible in clean plain text or simple LaTeX.

OUTPUT FORMAT (use these exact headings in this order):
1. ACTIVITY TITLE
2. CONCEPT REVEALED
3. PAPER AND TOOLS
4. FOLD-BY-FOLD STEPS (numbered; say which edges/points meet)
5. WHAT EACH FOLD SHOWS
6. WORKED EXAMPLE WITH TWO CHECKS
7. CHECK QUESTIONS (with answers)
8. TEACHER TIPS

RULES: Steps must be physically doable with ordinary paper. Be exact about which parts meet. Short, plain sentences.

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.

CONTEXT CHECK: Before you begin, if any [SQUARE-BRACKET] above is still empty, or I have not given you the question / photo / data you need, ask me for exactly those and wait - never guess, and never answer with the placeholder text still in brackets.

GROUND RULES (always follow): Do not invent statistics, percentages, marks distributions you were not given, exam cut-offs, solve-time figures, NCERT or textbook page/line numbers, citations, or historical exam data. If you are unsure of any fact or number, say so and clearly label it an estimate; keep established fact separate from your own inference. Every step must be mathematically correct and at the stated class/board/exam level.
