Previous-Year Trend Analysis for a Chapter (What Gets Asked)

An analysis of which sub-topics and question types a chapter usually asks, with a study priority.

Works on any free AI  ·  Best tool:  Any AI chat (ChatGPT/Gemini/Claude/Copilot)

✅ How to use this effectively

  1. Open any AI chat.
  2. Type the exam and chapter.
  3. Paste the prompt and send.
  4. Read the priority order.
  5. Plan teaching time around it.
  6. For a trustworthy answer key: paste the finished paper into a FRESH chat and ask it to solve and grade it as an unseen exam (it catches its own slips).
🔧 If it is not right, reply with this: If a claim feels like a guessed statistic, reply: "Don't state exact percentages as facts - rewrite the trends as 'typically asked / occasionally asked' and base priority on question-type frequency."
THE PROMPT
ROLE: Act as an exam pattern analyst who studies how a chapter is examined over the years.

CONTEXT: I teach [CHAPTER e.g. Rotational Motion / Probability / Coordinate Geometry] for [EXAM e.g. JEE Main / NEET] at [CLASS/GRADE] level. I want a clear picture of what this exam usually asks from this chapter, so I teach the right things.

YOUR TASK:
1. Break [CHAPTER] into its sub-topics and label how often each is typically examined: Very Frequent / Frequent / Occasional / Rare (based on standard exam patterns, described qualitatively).
2. List the question TYPES this chapter favours (e.g. single-correct, integer, assertion-reason) and the usual difficulty.
3. Identify 'evergreen' question ideas that recur and 'low-yield' areas safe to deprioritise.
4. Give a clear teaching/revision priority order with reasons.

INPUT: Here is the exam, chapter and class: [PASTE DETAILS HERE]. Stay inside the current syllabus for this class.

HOW TO WORK IT OUT: Base trends on well-established exam patterns. Do NOT fabricate exact year-by-year statistics or percentages; use qualitative frequency labels and explain the reasoning. If you are unsure whether something is frequent, say so rather than guessing a number. Never invent syllabus items. Keep any maths legible in clean LaTeX or clear plain text.

OUTPUT FORMAT, in this exact order:

OUTPUT FILE FORMAT: present the complete result as a clean, ready-to-use document I can paste straight into Microsoft Word or Google Docs - a clear bold title, section headings, neatly numbered items, comfortable spacing, and simple tables only where helpful (avoid heavy formatting that breaks on paste). It must also print cleanly to PDF on A4 (File > Print > Save as PDF). Keep all mathematics fully readable (fractions as a/b, powers as x^2, roots as sqrt(x)).

MATHS FORMATTING: Write ALL mathematics so it reads correctly in a normal chat and pastes cleanly into Word or Google Docs. Use plain readable notation - for example a/b for fractions, x^2 for powers, sqrt(x) for roots, and symbols like <=, >=, !=, +-, pi, theta, integral and sum. Do NOT output raw LaTeX or code such as \frac{}{}, \sqrt{}, x^{2}, $...$ or \( ... \) - these appear as broken symbols to a teacher in a free chat.
A. Title block: 'Exam: [EXAM]', 'Chapter: [CHAPTER]', 'Trend Analysis', 'Compiled by Indrajeet Yadav'.
B. SECTION 1 - SUB-TOPIC FREQUENCY: table Sub-topic | Frequency label | Why.
C. SECTION 2 - FAVOURED QUESTION TYPES: with usual difficulty.
D. SECTION 3 - EVERGREEN vs LOW-YIELD.
E. SECTION 4 - PRIORITY ORDER: ranked list of what to teach/revise first and why.

RULES: Qualitative frequency only, no fabricated stats. Stay in syllabus. Be honest about uncertainty.

At the very end add a footer line: 'Prepared by Indrajeet Yadav', and put 'Compiled by Indrajeet Yadav' in the title block.
